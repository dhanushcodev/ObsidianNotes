# 📌 @RestControllerAdvice in Spring Boot

## ✅ What is `@RestControllerAdvice`?

`@RestControllerAdvice` is a specialized annotation in Spring Boot used for **global exception handling** across all `@RestController` classes.

It is a combination of:

- `@ControllerAdvice` → applies globally to controllers
- `@ResponseBody` → returns JSON/XML instead of a view

👉 It is mainly used to **handle exceptions and return consistent API responses**.

---

## 🔥 Why use `@RestControllerAdvice`?

### ❌ Without it:

- You write `try-catch` in every controller
- Code becomes repetitive and messy

### ✅ With it:

- Centralized exception handling
- Clean controllers
- Consistent error response format

---

## 🧠 Basic Flow

1. Exception occurs in Controller
2. Spring looks for matching `@ExceptionHandler`
3. `@RestControllerAdvice` handles it
4. Returns structured JSON response

---

## 🧾 Example

### 📌 Step 1: Create Custom Exception

```java
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}
```

---

### 📌 Step 2: Use Exception in Controller

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping("/{id}")
    public String getUser(@PathVariable int id) {
        if (id != 1) {
            throw new UserNotFoundException("User not found with id: " + id);
        }
        return "User Found";
    }
}
```

---

### 📌 Step 3: Global Exception Handler

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<Map<String, String>> handleUserNotFound(UserNotFoundException ex) {

        Map<String, String> error = new HashMap<>();
        error.put("error", ex.getMessage());
        error.put("status", "404");

        return new ResponseEntity<>(error, HttpStatus.NOT_FOUND);
    }
}
```

---

## 📦 Output (JSON Response)

### Request

```
GET /users/5
```

### Response

```json
{
  "error": "User not found with id: 5",
  "status": "404"
}
```

## 🎯 Handling Multiple Exceptions

```java
@ExceptionHandler(Exception.class)
public ResponseEntity<String> handleAllExceptions(Exception ex) {
    return new ResponseEntity<>("Something went wrong", HttpStatus.INTERNAL_SERVER_ERROR);
}
```

---

## 🎯 Handling Validation Errors

```java
@ExceptionHandler(MethodArgumentNotValidException.class)
public ResponseEntity<Map<String, String>> handleValidation(MethodArgumentNotValidException ex) {

    Map<String, String> errors = new HashMap<>();

    ex.getBindingResult().getFieldErrors().forEach(error -> {
        errors.put(error.getField(), error.getDefaultMessage());
    });

    return new ResponseEntity<>(errors, HttpStatus.BAD_REQUEST);
}
```

---

## ⚡ Key Annotations

|Annotation|Purpose|
|---|---|
|`@RestControllerAdvice`|Global exception handler|
|`@ExceptionHandler`|Handles specific exception|
|`ResponseEntity`|Customize response|

---

## 🚀 Best Practices

- Create **custom exception classes**
- Use **consistent response format** (error, timestamp, status)
- Handle **specific exceptions first**, then generic ones
- Avoid exposing internal error details

---

## 🧠 Difference: `@ControllerAdvice` vs `@RestControllerAdvice`

|Feature|ControllerAdvice|RestControllerAdvice|
|---|---|---|
|Response Type|View (HTML)|JSON/XML|
|Includes `@ResponseBody`|❌|✅|
|Used for REST APIs|❌|✅|

---

## 🧩 Summary

- `@RestControllerAdvice` = Global exception handler for REST APIs
- Removes repetitive try-catch
- Keeps controllers clean
- Returns structured JSON error responses
