
## 🔍 What is JPA Auditing in Spring Boot?

**JPA Auditing** is a feature in Spring Boot that automatically tracks and populates metadata like:

- Who created the record  
- When it was created  
- Who last modified it  
- When it was last updated  

This is extremely useful for logging, tracking changes, and maintaining data history.

---

## ⚙️ 1. Enable JPA Auditing

To use auditing, first enable it in your main Spring Boot application:

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.data.jpa.repository.config.EnableJpaAuditing;

@SpringBootApplication
@EnableJpaAuditing
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
````

---

## 🧩 2. Add Auditing Fields in Entity

Use annotations provided by Spring Data JPA:

```java
import jakarta.persistence.*;
import lombok.Getter;
import lombok.Setter;
import org.springframework.data.annotation.*;
import org.springframework.data.jpa.domain.support.AuditingEntityListener;

import java.time.LocalDateTime;

@Getter
@Setter
@Entity
@EntityListeners(AuditingEntityListener.class)
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    @CreatedDate
    private LocalDateTime createdDate;

    @LastModifiedDate
    private LocalDateTime lastModifiedDate;

    @CreatedBy
    private String createdBy;

    @LastModifiedBy
    private String lastModifiedBy;
}
```

---

## 🧠 3. Provide Auditor (Who is the User?)

To populate `@CreatedBy` and `@LastModifiedBy`, implement `AuditorAware`:

```java
import org.springframework.data.domain.AuditorAware;
import org.springframework.stereotype.Component;

import java.util.Optional;

@Component
public class AuditorAwareImpl implements AuditorAware<String> {

    @Override
    public Optional<String> getCurrentAuditor() {
        // In real apps, fetch from Spring Security context
        return Optional.of("SYSTEM_USER");
    }
}
```

---

## 🔗 4. Configure Auditor Bean (Optional if using @Component)

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.domain.AuditorAware;

@Configuration
public class AuditConfig {

    @Bean
    public AuditorAware<String> auditorProvider() {
        return new AuditorAwareImpl();
    }
}
```

## 🏗️ Best Practice: Use a Base Entity

Instead of repeating fields, create a base class:

```java
import jakarta.persistence.*;
import lombok.Getter;
import lombok.Setter;
import org.springframework.data.annotation.*;
import org.springframework.data.jpa.domain.support.AuditingEntityListener;

import java.time.LocalDateTime;

@Getter
@Setter
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class Auditable {

    @CreatedDate
    private LocalDateTime createdDate;

    @LastModifiedDate
    private LocalDateTime lastModifiedDate;

    @CreatedBy
    private String createdBy;

    @LastModifiedBy
    private String lastModifiedBy;
}
```

### Then extend it:

```java
@Entity
public class User extends Auditable {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
}
```

---

## 🔑 Key Annotations Summary

|Annotation|Purpose|
|---|---|
|`@CreatedDate`|Set creation timestamp|
|`@LastModifiedDate`|Set last update timestamp|
|`@CreatedBy`|Set creator|
|`@LastModifiedBy`|Set last modifier|
|`@EnableJpaAuditing`|Enable auditing feature|
|`@EntityListeners`|Hook auditing into entity|

## 🚀 Real-World Use Cases

- Tracking who created/updated records
- Audit logs for compliance
- Debugging data issues
- Multi-user systems with change history

---

## ⚠️ Important Notes

- Works only when saving via **Spring Data JPA (Repository)**
- Requires **entity listener** to be enabled
- `@CreatedBy` needs `AuditorAware` implementation
- Works well with **Spring Security** for real users
