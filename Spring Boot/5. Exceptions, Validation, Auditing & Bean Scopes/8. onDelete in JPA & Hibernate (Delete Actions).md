## 🔹 What is @OnDelete?

@OnDelete is used to define what happens to child records when a parent record is deleted.

It is a Hibernate-specific annotation:

@OnDelete(action = OnDeleteAction.CASCADE)

👉 Works at the database level (foreign key constraint)

---

## 🔹 Why do we need it?

Without onDelete:
- Deleting parent may fail due to FK constraint
- Orphan records may remain
- Manual deletion may be required

---

## 🔹 Types of Delete Actions

1. CASCADE  
2. SET NULL  
3. RESTRICT / NO ACTION (default)

---

## 🔹 1. CASCADE

👉 When parent is deleted → all child records are also deleted

### Example

Parent Entity:

```java
@Entity
public class Department {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToMany(mappedBy = "department")
    @OnDelete(action = OnDeleteAction.CASCADE)
    private List<Employee> employees;
}
```


Child Entity:

```java
@Entity
public class Employee {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @ManyToOne
    @JoinColumn(name = "department_id")
    private Department department;
}
```

### Behavior

`departmentRepository.deleteById(1L);`

👉 All employees under that department are deleted

---

## 🔹 2. SET NULL

👉 When parent is deleted → child foreign key becomes NULL

### Example

```java
@ManyToOne
@JoinColumn(name = "department_id", nullable = true)
@OnDelete(action = OnDeleteAction.SET_NULL)
private Department department;
```

### Behavior

- Employees remain
- department_id becomes NULL

---

## 🔹 3. RESTRICT / NO ACTION (Default)

👉 Prevents deletion if child exists

### Example

`departmentRepository.delete(department);`

👉 Throws:
`ConstraintViolationException`

---

## 🔹 Difference: CascadeType.REMOVE vs @OnDelete

| Feature | CascadeType.REMOVE | @OnDelete |
|--------|------------------|----------|
| Level | Hibernate (ORM) | Database (FK) |
| Execution | Multiple queries | Single DB operation |
| Performance | Slower | Faster |
| Lifecycle Events | Triggered | Not triggered |

---

## 🔹 Important Notes

- Works only with Hibernate
- Requires DB foreign key constraints
- Does NOT trigger Hibernate lifecycle callbacks
- Best for performance and bulk deletes

---

## 🔹 Real-World Example

User → Orders

- CASCADE → delete all orders  
- SET NULL → keep orders, remove user reference  
- RESTRICT → block deletion  

---

## 🔹 When to Use What?

| Scenario | Recommended |
|--------|------------|
| Strong dependency (Order → Items) | CASCADE |
| Optional relation (Employee → Department) | SET NULL |
| Critical data protection | RESTRICT |

---

## 🔹 Summary

- @OnDelete controls DB-level delete behavior
- CASCADE → deletes children
- SET NULL → nullifies foreign key
- RESTRICT → prevents deletion
- Faster than CascadeType.REMOVE but less flexible
