# 🧩 JPA Mappings (Concept + Explanation + Example)

JPA mappings define:

👉 How Java objects ↔ Database tables  
👉 How entities relate to each other

---

# 🔹 1. Basic Mapping (@Column)

## ✅ Concept

Maps a Java field → database column

## ✅ Example

```java
@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "user_name", nullable = false, length = 50)
    private String name;

    @Column(unique = true)
    private String email;
}
```

## 🔍 Explanation

- `@Entity` → marks class as table
- `@Table` → custom table name
- `@Column` → customize column properties
- Default: field name = column name

# 🔹 2. @OneToOne Mapping

## ✅ Concept

One entity ↔ One entity

👉 Example: User ↔ Profile

## ✅ Example

```java
@Entity
public class User {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToOne
    @JoinColumn(name = "profile_id")
    private Profile profile;
}
```

```java
@Entity
public class Profile {

    @Id
    @GeneratedValue
    private Long id;

    private String bio;
}
```

## 🔍 Explanation

- `@JoinColumn` creates foreign key
- FK stored in **User table**

```
User:
id | name | profile_id (FK)

Profile:
id | bio
```

- Owning side = class with `@JoinColumn`

---

# 🔹 3. @OneToMany Mapping

## ✅ Concept

One → Many

👉 Example: User → Orders

## ✅ Example

```java
@Entity
public class User {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
    private List<Order> orders;
}
```

```java
@Entity
public class Order {

    @Id
    @GeneratedValue
    private Long id;

    private String product;

    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}
```

## 🔍 Explanation

- `mappedBy` → Order owns relationship
- FK stored in **Order table**

```
Order:
id | product | user_id (FK)
```

- Always use bidirectional mapping for efficiency

---

# 🔹 4. @ManyToOne Mapping

## ✅ Concept

Many → One

👉 Example: Many Orders → One User

## 🔍 Example

```java
@ManyToOne
@JoinColumn(name = "user_id")
private User user;
```

## 🔍 Explanation

- Creates foreign key
- Most commonly used mapping
- Efficient (no extra table)

---

# 🔹 5. @ManyToMany Mapping

## ✅ Concept

Many ↔ Many

👉 Example: Student ↔ Course

## ✅ Example

```java
@Entity
public class Student {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @ManyToMany
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private List<Course> courses;
}
```

## 🔍 Explanation

- Uses join table

```
student_course:
student_id | course_id
```

## ⚠️ Problem

- Cannot add extra fields
- Hard to maintain

👉 ✅ Use join entity instead in real projects

---

# 🔹 6. @Embedded & @Embeddable

## ✅ Concept

Reuse object inside entity

👉 Example: Address inside User

## ✅ Example

```java
@Embeddable
public class Address {
    private String city;
    private String state;
}
```

```java
@Entity
public class User {

    @Id
    private Long id;

    @Embedded
    private Address address;
}
```

## 🔍 Explanation

- No separate table
- Fields stored in same table

```
User:
id | city | state
```

# 🔹 7. Inheritance Mapping

## ✅ Concept

Maps OOP inheritance → DB

## ✅ Example

```java
@Entity
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
public class Vehicle {
    @Id
    private Long id;

    private String type;
}
```

```java
@Entity
public class Car extends Vehicle {
    private String model;
}
```

## 🔍 Strategies

### SINGLE_TABLE

- One table for all classes (fastest)

```
Vehicle:
id | type | model
```

### JOINED

- Multiple tables with JOIN

### TABLE_PER_CLASS

- Separate table per class

# 🔥 Important Concepts

## 🧠 1. Owning vs Inverse Side

- Owning side → has FK, controls DB
- Inverse side → uses `mappedBy`

```java
@OneToMany(mappedBy = "user") // inverse
@ManyToOne // owning
```

---

## 🧠 2. Cascade Types

```java
cascade = CascadeType.ALL
```

### Types:

- PERSIST
- MERGE
- REMOVE
- ALL

👉 Saving parent saves children

---

## 🧠 3. Fetch Types

```java
fetch = FetchType.LAZY
fetch = FetchType.EAGER
```

### LAZY

- Load when needed

### EAGER

- Load immediately

---

# 🔥 Best Practices

✅ Prefer:

- `@ManyToOne` over `@OneToMany`

❌ Avoid:

- Direct `@ManyToMany`

✅ Always:

- Use `mappedBy`
- Use LAZY loading

---

# 🚀 Final Mental Model

👉 Where is FK? → Owning side  
👉 Who updates DB? → Owning side  
👉 Need extra fields? → Avoid ManyToMany
