# 🌱 Bean Scopes in Spring Boot

In Spring Boot (Spring Framework), a **bean scope** defines:

- How many instances of a bean are created
- How long the bean lives in the Spring container

---

# 📌 1. Singleton Scope (Default)

## 🔹 Definition

- Only **one instance** per Spring container
- Shared across the entire application

## 🔹 Example

```java
@Component
@Scope("singleton") // optional (default)
public class MyService {
}
```

## 🔹 Behavior

```java
@Autowired
MyService s1;

@Autowired
MyService s2;

// s1 == s2 → true
```

## 🔹 Use Case

- Stateless services
- Utility classes

---

# 📌 2. Prototype Scope

## 🔹 Definition

- A **new instance** is created every time the bean is requested

## 🔹 Example

```java
@Component
@Scope("prototype")
public class MyPrototypeService {
}
```

## 🔹 Behavior

```java
@Autowired
ApplicationContext context;

MyPrototypeService p1 = context.getBean(MyPrototypeService.class);
MyPrototypeService p2 = context.getBean(MyPrototypeService.class);

// p1 == p2 → false
```

## 🔹 Use Case

- Stateful objects
- Temporary processing objects

---

# 📌 3. Request Scope (Web Only)

## 🔹 Definition

- One bean instance per **HTTP request**

## 🔹 Example

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class RequestBean {
}
```

## 🔹 Behavior

- New instance created for every HTTP request

## 🔹 Use Case

- Request-specific data

---

# 📌 4. Session Scope (Web Only)

## 🔹 Definition

- One bean instance per **HTTP session (per user)**

## 🔹 Example

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class SessionBean {
}
```

## 🔹 Behavior

- Same instance reused across multiple requests of same user

## 🔹 Use Case

- User session data (e.g., shopping cart)

---

# 📌 5. Application Scope (Web Only)

## 🔹 Definition

- One bean per **ServletContext (entire application)**

## 🔹 Example

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_APPLICATION)
public class AppBean {
}
```

## 🔹 Behavior

- Shared across the whole application

## 🔹 Use Case

- Global shared data

---

# 📌 6. WebSocket Scope

## 🔹 Definition

- One bean per **WebSocket session**

## 🔹 Example

```java
@Component
@Scope("websocket")
public class WebSocketBean {
}
```

## 🔹 Use Case

- Real-time applications (chat, notifications)

---

# ⚖️ Comparison Table

|Scope|Instances Created|Lifecycle|Use Case|
|---|---|---|---|
|Singleton|1|Application lifetime|Stateless services|
|Prototype|Many|On demand|Stateful objects|
|Request|Per request|HTTP request|Request-specific data|
|Session|Per session|User session|User-specific data|
|Application|1 per app|App lifecycle|Global shared data|
|WebSocket|Per WS session|WebSocket lifecycle|Real-time communication|

---

# 🚀 Important Points

- Default scope = **Singleton**
- Prototype beans are **not fully managed** after creation
- Injecting Prototype into Singleton → behaves like Singleton (unless using `ObjectFactory` or `Provider`)
- Web scopes require **Spring Web context**
