# 🌱 Bean Scopes in Spring Boot (Complete Notes)

In Spring Boot (Spring Framework), a **bean scope** defines:

* How many instances of a bean are created
* How long the bean lives in the Spring container

---

# 📌 1. Singleton Scope (Default)

## 🔹 Definition

* Only **one instance** per Spring container
* Shared across the entire application

## 🔹 Example

```java
@Component
@Scope("singleton") // optional (default)
public class MyService {
}
```

## 🔹 Behavior

```java
@Autowired
MyService s1;

@Autowired
MyService s2;

// s1 == s2 → true
```

## 🔹 Use Case

* Stateless services
* Utility classes

---

# 📌 2. Prototype Scope

## 🔹 Definition

* A **new instance** is created every time the bean is requested

## 🔹 Example

```java
@Component
@Scope("prototype")
public class MyPrototypeService {
}
```

## 🔹 Behavior

```java
@Autowired
ApplicationContext context;

MyPrototypeService p1 = context.getBean(MyPrototypeService.class);
MyPrototypeService p2 = context.getBean(MyPrototypeService.class);

// p1 == p2 → false
```

## 🔹 Use Case

* Stateful objects
* Temporary processing objects

---

# 📌 3. Request Scope (Web Only)

## 🔹 Definition

* One bean instance per **HTTP request**

## 🔹 Two Ways to Define

### ✅ Using @Scope

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class RequestBean {
}
```

### ✅ Using Shortcut Annotation (Recommended)

```java
@Component
@RequestScope
public class RequestBean {

    private final String requestId = UUID.randomUUID().toString();

    public String getRequestId() {
        return requestId;
    }
}
```

## 🔹 Controller Example

```java
@RestController
public class DemoController {

    @Autowired
    private RequestBean requestBean;

    @GetMapping("/request")
    public String getRequest() {
        return "Request ID: " + requestBean.getRequestId();
    }
}
```

## 🔹 Behavior

* Every API call → **new object**

---

# 📌 4. Session Scope (Web Only)

## 🔹 Definition

* One bean instance per **HTTP session (per user)**

## 🔹 Two Ways to Define

### ✅ Using @Scope

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class SessionBean {
}
```

### ✅ Using Shortcut Annotation (Recommended)

```java
@Component
@SessionScope
public class SessionBean {

    private int counter = 0;

    public int increment() {
        return ++counter;
    }
}
```

## 🔹 Controller Example

```java
@RestController
public class DemoController {

    @Autowired
    private SessionBean sessionBean;

    @GetMapping("/session")
    public String getSession() {
        return "Session Counter: " + sessionBean.increment();
    }
}
```

## 🔹 Behavior

* Same user → same object
* New user → new object

---

# 📌 5. Application Scope (Web Only)

## 🔹 Definition

* One bean per **entire application (ServletContext)**

## 🔹 Two Ways to Define

### ✅ Using @Scope

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_APPLICATION)
public class AppBean {
}
```

### ✅ Using Shortcut Annotation (Recommended)

```java
@Component
@ApplicationScope
public class AppBean {

    private int globalCounter = 0;

    public int increment() {
        return ++globalCounter;
    }
}
```

## 🔹 Controller Example

```java
@RestController
public class DemoController {

    @Autowired
    private AppBean appBean;

    @GetMapping("/app")
    public String getApp() {
        return "Global Counter: " + appBean.increment();
    }
}
```

## 🔹 Behavior

* Shared across all users

---

# 📌 6. WebSocket Scope

## 🔹 Definition

* One bean per **WebSocket session**

## 🔹 Example

```java
@Component
@Scope("websocket")
public class WebSocketBean {
}
```

## 🔹 Use Case

* Real-time apps (chat, notifications)

---

# ⚖️ Comparison Table

| Scope       | Instances Created | Lifecycle            | Use Case                |
| ----------- | ----------------- | -------------------- | ----------------------- |
| Singleton   | 1                 | Application lifetime | Stateless services      |
| Prototype   | Many              | On demand            | Stateful objects        |
| Request     | Per request       | HTTP request         | Request-specific data   |
| Session     | Per session       | User session         | User-specific data      |
| Application | 1 per app         | App lifecycle        | Global shared data      |
| WebSocket   | Per WS session    | WebSocket lifecycle  | Real-time communication |

---

# 🔥 Real-Time Difference (Easy Understanding)

| Scenario         | RequestScope   | SessionScope | ApplicationScope |
| ---------------- | -------------- | ------------ | ---------------- |
| Refresh same API | New object     | Same object  | Same object      |
| Different user   | New object     | New object   | Same object      |
| Data sharing     | No             | Per user     | Global           |
| Lifecycle        | Single request | Session      | App lifetime     |

---

# 🚀 Important Interview Points

* Default scope = **Singleton**
* `@RequestScope`, `@SessionScope`, `@ApplicationScope` are **shortcuts of @Scope**
* Prototype beans are **not fully managed** after creation
* Injecting Prototype into Singleton → behaves like Singleton (unless using `ObjectFactory` or `Provider`)
* Web scopes require **Spring Web context**
* Shortcut annotations automatically handle **proxyMode**

---

# ✅ When to Use

* Singleton → services, repositories
* Prototype → temporary/stateful objects
* Request → request-specific data
* Session → user data (cart, login)
* Application → global shared data
* WebSocket → real-time session data
