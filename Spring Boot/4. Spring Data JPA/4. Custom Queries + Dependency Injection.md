# 📘 Custom Query Methods (Method Naming)

Spring Data JPA generates queries automatically based on method names.

---

## 🔹 Example

```java
List<User> findByName(String name);
User findByEmail(String email);
List<User> findByNameAndEmail(String name, String email);
```

---

## 🔹 Internally

```sql
SELECT * FROM users WHERE name = ?
```

---

## 🔹 Keywords

| Keyword        | Example                        | SQL Equivalent        |
|----------------|--------------------------------|----------------------|
| `And`          | findByNameAndEmail            | WHERE name AND email |
| `Or`           | findByNameOrEmail             | WHERE name OR email  |
| `Between`      | findBySalaryBetween           | BETWEEN              |
| `LessThan`     | findByAgeLessThan             | <                    |
| `GreaterThan`  | findByAgeGreaterThan          | >                    |
| `Like`         | findByNameLike                | LIKE                 |
| `In`           | findByIdIn                    | IN (...)             |

# 📘 JPQL (Java Persistence Query Language)

## 🔹 What is JPQL?

JPQL is a **query language that works on Entity objects, NOT database tables**.

👉 It uses:
- Entity class names
- Field names (not column names)

---

## 🔹 Key Difference: JPQL vs SQL

| Feature        | JPQL                          | SQL                     |
|----------------|-------------------------------|--------------------------|
| Works on       | Entity (Java class)           | Tables (DB)              |
| Uses           | Field names                   | Column names             |
| Example        | `User`                        | `users`                  |

---

## 🔹 Example Entity

```java
@Entity
@Table(name = "users")
public class User {
    private String name;
    private String email;
}
```

---

## 🔹 JPQL Query Example

```java
@Query("SELECT u FROM User u WHERE u.name = :name")
List<User> getUsersByName(@Param("name") String name);
```

---

## 🔹 How It Works

JPQL:
```sql
SELECT u FROM User u WHERE u.name = :name
```

Converted by Hibernate into SQL:
```sql
SELECT * FROM users WHERE name = ?
```

---

## 🔹 JPQL Features

- Uses **Entity name (`User`)**
- Uses **object alias (`u`)**
- Uses **field names (`name`)**
- Supports **joins, sorting, grouping**

---

## 🔹 JPQL with Multiple Conditions

```java
@Query("SELECT u FROM User u WHERE u.name = :name AND u.email = :email")
User getUser(@Param("name") String name, @Param("email") String email);
```

---

## 🔹 JPQL vs Native Query

### JPQL
```java
@Query("SELECT u FROM User u WHERE u.email = :email")
```

### Native SQL
```java
@Query(value = "SELECT * FROM users WHERE email = :email", nativeQuery = true)
```

# 📘 Repository Example

```java
package com.example.demo.repository;

import com.example.demo.entity.User;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

import java.util.List;

public interface UserRepository extends JpaRepository<User, Long> {

    // Derived query
    List<User> findByName(String name);

    // JPQL query
    @Query("SELECT u FROM User u WHERE u.email = :email")
    User getUserByEmail(@Param("email") String email);
}
```

# 📘 Dependency Injection (How Repository Works)

## ❓ Is UserRepository an object?

👉 YES — but created by Spring automatically.

---

## 🔹 Service Example

```java
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

---

## 🔹 What Happens Internally?

1. You define interface:
```java
public interface UserRepository extends JpaRepository<User, Long> {}
```

2. Spring:
   - Creates implementation (`SimpleJpaRepository`)
   - Wraps it with proxy
   - Registers as Bean

3. Injects into Service

---

## 🔹 Internal Representation

```java
UserRepository repo = new SimpleJpaRepository(...);
```

👉 (Handled by Spring automatically)

---

## 🔹 Why No Implementation Needed?

Because Spring uses:
- Reflection  
- Dynamic Proxy  
- AOP

# 📘 Full Example

---

## Entity

```java
@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private String email;
}
```

---

## Repository

```java
public interface UserRepository extends JpaRepository<User, Long> {

    List<User> findByName(String name);

    @Query("SELECT u FROM User u WHERE u.email = :email")
    User getUserByEmail(@Param("email") String email);
}
```

---

## Service

```java
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public User save(User user) {
        return userRepository.save(user);
    }

    public List<User> getAll() {
        return userRepository.findAll();
    }
}
```

---

## Controller

```java
@RestController
@RequestMapping("/users")
public class UserController {

    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @PostMapping
    public User create(@RequestBody User user) {
        return userService.save(user);
    }

    @GetMapping
    public List<User> getAll() {
        return userService.getAll();
    }
}
```

---

# 🎯 Final Summary

- `JpaRepository` simplifies DB operations  
- Custom queries:
  - Method naming
  - `@Query` (JPQL / Native SQL)  
- JPQL works on **entities, not tables**  
- Spring automatically:
  - Creates repository object
  - Injects into service  
- No manual implementation needed  

---

# 🧠 Interview Answer

👉 *"JPQL is an object-oriented query language that works on entity classes instead of database tables. Spring Data JPA uses it with @Query to define custom queries, which are internally converted into SQL by Hibernate."*
