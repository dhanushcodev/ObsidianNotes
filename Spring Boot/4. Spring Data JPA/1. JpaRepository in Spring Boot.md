# 📘 JpaRepository in Spring Boot (Detailed Explanation)

---

## 🔹 What is JpaRepository?

`JpaRepository` is an **interface provided by Spring Data JPA** that helps you perform **database operations (CRUD + advanced features)** without writing SQL queries.

📦 Package:
```java
org.springframework.data.jpa.repository.JpaRepository
```

👉 It internally uses **JPA (Java Persistence API)** and commonly **Hibernate** as the implementation.

---

## 🔹 Hierarchy (Very Important)

```
Repository
   ↑
CrudRepository
   ↑
PagingAndSortingRepository
   ↑
JpaRepository
```

👉 This means `JpaRepository` includes:
- CRUD operations  
- Pagination & sorting  
- JPA-specific features  

---

## 🔹 How to Use JpaRepository

### 1️⃣ Create Entity

```java
@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private String email;
}
```

---

### 2️⃣ Create Repository Interface

```java
public interface UserRepository extends JpaRepository<User, Long> {
}
```

### 🔸 Explanation:
- `User` → Entity class  
- `Long` → Primary key type  

---

## 🔹 Built-in Methods

### ✅ CRUD Operations

```java
save(entity);         // insert or update
findById(id);        // get by id
findAll();           // get all records
deleteById(id);      // delete by id
existsById(id);      // check existence
count();             // count records
```

---

### ✅ Batch Operations

```java
saveAll(List<T> entities);
deleteAll();
deleteAllInBatch();
```

---

### ✅ Pagination & Sorting

```java
findAll(Pageable pageable);
findAll(Sort sort);
```

Example:

```java
Pageable pageable = PageRequest.of(0, 5);
userRepository.findAll(pageable);
```

---

### ✅ JPA-Specific Methods

```java
flush();                     // sync persistence context with DB
saveAndFlush(entity);       // save + flush immediately
deleteInBatch(entities);    // batch delete
```

---

## 🔹 Custom Query Methods (No SQL Needed)

Spring generates queries based on method names.

### Example:

```java
List<User> findByName(String name);
User findByEmail(String email);
List<User> findByNameAndEmail(String name, String email);
```

👉 Internally converted to SQL like:
```sql
SELECT * FROM users WHERE name = ?
```

---

## 🔹 Custom Queries using @Query

### JPQL (Recommended)

```java
@Query("SELECT u FROM User u WHERE u.name = :name")
List<User> getUsersByName(@Param("name") String name);
```

---

### Native SQL

```java
@Query(value = "SELECT * FROM users WHERE email = :email", nativeQuery = true)
User getByEmail(@Param("email") String email);
```

## 🔹 Example Service Usage

```java
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public User saveUser(User user) {
        return userRepository.save(user);
    }

    public List<User> getAllUsers() {
        return userRepository.findAll();
    }
}
```

---

## 🔹 How JpaRepository Works Internally

When you define:

```java
public interface UserRepository extends JpaRepository<User, Long> {}
```

👉 Spring Boot automatically:

1. Creates an implementation class (`SimpleJpaRepository`)  
2. Wraps it in a proxy  
3. Registers it as a **Spring Bean**  
4. Injects it wherever needed  

---

## 🔹 Dependency Injection (Important)

```java
public UserService(UserRepository userRepository) {
    this.userRepository = userRepository;
}
```

👉 Yes, `UserRepository` is passed as an object —  
but you **never create it manually**.

✔ Spring creates and injects it automatically.

## 🔹 Advantages

- ✅ No SQL required  
- ✅ Less boilerplate code  
- ✅ Built-in powerful methods  
- ✅ Easy pagination & sorting  
- ✅ Clean architecture  
- ✅ Production-ready  

---

## 🔹 When to Use JpaRepository?

Use it when:
- Working with **relational databases**
- You want **fast development**
- You prefer **minimal SQL writing**
- You need **pagination, sorting, batch operations**

---

## 🔹 JpaRepository vs CrudRepository

| Feature              | CrudRepository | JpaRepository |
|---------------------|---------------|--------------|
| CRUD operations     | ✅            | ✅           |
| Pagination          | ❌            | ✅           |
| Sorting             | ❌            | ✅           |
| Batch operations    | ❌            | ✅           |
| JPA-specific methods| ❌            | ✅           |

---

## 🔹 Key Points to Remember

- You NEVER implement `JpaRepository` manually  
- Spring generates implementation at runtime  
- Supports both **derived queries** and **custom queries**  
- Works with **Hibernate under the hood**  

---

## 🎯 Final Summary

- `JpaRepository` is a **powerful abstraction for database access**  
- Extends multiple repository interfaces  
- Provides ready-to-use methods + query generation  
- Reduces manual coding effort significantly  
- Core part of modern Spring Boot applications
