# рЯФє Hibernate `ddl-auto` Explained

In **Hibernate ORM**, the `ddl-auto` property is used to control how Hibernate handles **database schema (DDL = Data Definition Language)** operations automatically.

It is commonly configured in **Spring Boot** using:

```properties
spring.jpa.hibernate.ddl-auto=VALUE
```

---

## рЯФС Available `ddl-auto` Options

### 1. `none`

- Hibernate **does nothing**
- No table creation, update, or validation

вЬЕ Use when:

- You manage DB manually (SQL scripts, Flyway, Liquibase)

---

### 2. `validate`

- Checks if DB schema **matches your entities**
- вЭМ If mismatch вЖТ throws error

вЬЕ Use when:

- Production environment (safe option)

---

### 3. `update`

- Updates schema **without deleting data**
- Adds missing tables/columns
- Add tables / columns from entities if missing

вЪ†пЄП Limitations:

- Does NOT remove unused columns
- May cause unexpected issues in production

вЬЕ Use when:

- Development phase

---

### 4. `create`

- Drops existing tables and **creates new ones**
- Data is **lost**

вЬЕ Use when:

- Testing / initial setup

---

### 5. `create-drop`

- Same as `create`
- Drops tables again when app stops

вЬЕ Use when:

- Unit testing

---

## рЯУК Summary Table

|Value|Behavior|Data Safety|Usage|
|---|---|---|---|
|none|Do nothing|вЬЕ Safe|Production|
|validate|Check schema|вЬЕ Safe|Production|
|update|Modify schema (no drop)|вЪ†пЄП Medium|Development|
|create|Drop & recreate schema|вЭМ Unsafe|Testing|

## рЯТ° Best Practices

- вЬЕ **Production** вЖТ `validate` or `none`
- вЬЕ **Development** вЖТ `update`
- вЬЕ **Testing** вЖТ `create` / `create-drop`
- вЭМ Avoid `update` in production (can cause silent issues)

## рЯІ† Simple Analogy

- `none` вЖТ "I won't touch anything"
