## рЯФє What is Lombok?

**Lombok** is a Java library that helps you **reduce boilerplate code** (like getters, setters, constructors, `toString`, etc.) by using simple annotations.

Instead of writing repetitive code manually, Lombok generates it **at compile time**.

---

## рЯФє Why use Lombok?

Without Lombok, a simple class looks like this:

```java
public class User {
    private String name;
    private int age;

    public User() {}

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}
```

рЯСЙ Too much boilerplate!

---

## рЯФє Same code using Lombok

```java
import lombok.*;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private String name;
    private int age;
}
```

вЬЕ ThatвАЩs it вАФ Lombok generates everything automatically.

## рЯФє Common Lombok Annotations

### 1. `@Getter` and `@Setter`

```java
@Getter
@Setter
public class User {
    private String name;
}
```

рЯСЙ Generates getter & setter methods.

---

### 2. `@ToString`

```java
@ToString
public class User {
    private String name;
}
```

рЯСЙ Generates `toString()` method.

---

### 3. `@EqualsAndHashCode`

```java
@EqualsAndHashCode
public class User {
    private String name;
}
```

рЯСЙ Generates `equals()` and `hashCode()`.

---

### 4. `@NoArgsConstructor`, `@AllArgsConstructor`, `@RequiredArgsConstructor`

```java
@NoArgsConstructor
@AllArgsConstructor
@RequiredArgsConstructor
public class User {
    private final String name;
    private int age;
}
```

рЯСЙ Generates constructors.

---

### 5. `@Data` (Most Popular)

```java
@Data
public class User {
    private String name;
    private int age;
}
```

рЯСЙ Combines:

- `@Getter`
    
- `@Setter`
    
- `@ToString`
    
- `@EqualsAndHashCode`
    
- `@RequiredArgsConstructor`
    

---

### 6. `@Builder` (Very Useful)

```java
import lombok.Builder;

@Builder
public class User {
    private String name;
    private int age;
}
```

Usage:

```java
User user = User.builder()
                .name("Dhanush")
                .age(25)
                .build();
```

рЯСЙ Clean object creation (no messy constructors).

---

### 7. `@Slf4j` (Logging)

```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
public class Demo {
    public void test() {
        log.info("Hello from Lombok!");
    }
}
```

рЯСЙ Automatically creates a logger.

---

## рЯФє Real Example in Spring Boot

```java
import lombok.Data;

@Data
public class UserDTO {
    private String username;
    private String password;
}
```

```java
@RestController
public class UserController {

    @PostMapping("/user")
    public UserDTO createUser(@RequestBody UserDTO user) {
        return user;
    }
}
```

рЯСЙ No need to write getters/setters manually.

## рЯФє Real-World Use Cases

### вЬЕ 1. DTOs (Data Transfer Objects)

- Reduce code for request/response objects
    
- Very common in Spring Boot APIs
    

---

### вЬЕ 2. JPA Entities

```java
@Entity
@Data
public class User {
    @Id
    private Long id;
    private String name;
}
```

рЯСЙ Eliminates boilerplate in entities

вЪ†пЄП Usually better to avoid `@Data` and use selective annotations (`@Getter`, `@Setter`) in entities.

---

### вЬЕ 3. Builder Pattern

```java
User user = User.builder()
                .name("Dhanush")
                .age(25)
                .build();
```

рЯСЙ Useful when:

- Many fields
    
- Optional parameters
    

---

### вЬЕ 4. Logging

```java
@Slf4j
public class Service {
    public void process() {
        log.info("Processing...");
    }
}
```

рЯСЙ No manual logger setup needed

---

### вЬЕ 5. Immutable Objects

```java
import lombok.Value;

@Value
public class User {
    String name;
    int age;
}
```

рЯСЙ Creates:

- Final fields
    
- No setters
    
- Immutable class

## рЯФє Advantages

- вЬЕ Reduces boilerplate code
    
- вЬЕ Improves readability
    
- вЬЕ Faster development
    
- вЬЕ Cleaner classes
    

---

## рЯФє Disadvantages

- вЪ†пЄП Hidden code (generated at compile time)
    
- вЪ†пЄП Can confuse beginners
    
- вЪ†пЄП IDE setup required (plugin needed)
    
- вЪ†пЄП Debugging sometimes tricky
    

---

## рЯФє When NOT to Use Lombok

- Very complex business logic classes
    
- When explicit code clarity is important
    
- In libraries where you want full transparency
