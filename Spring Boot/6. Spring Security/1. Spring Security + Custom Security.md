# 🔐 Spring Security + Custom Security (Clean Notes)

---

## 🧠 1. What is Spring Security?

Spring Security is a framework used to:

- **Authentication** → Verify user identity (username/password)
- **Authorization** → Control access (roles/permissions)

---

## 🔑 2. Authentication vs Authorization

|Type|Description|
|---|---|
|Authentication|Who are you?|
|Authorization|What can you access?|

---

## 🏗️ 3. Default Behavior

After adding dependency:

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

- All APIs are secured 🔐
- Default login page is shown
- Username: `user`
- Password: printed in console

---

## 🔐 4. Security Configuration (Core)

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {

        http
            .csrf(csrf -> csrf.disable()) // disable for REST APIs

            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public").permitAll()
                .requestMatchers("/admin").hasRole("ADMIN")
                .anyRequest().authenticated()
            )

            .httpBasic(Customizer.withDefaults())   // for Postman / APIs
            .formLogin(Customizer.withDefaults());  // for browser login

        return http.build();
    }
}
```

---

## 🧠 5. Key Concepts in Config

### 🔹 SecurityFilterChain

- Core security pipeline
- Every request passes through it

### 🔹 HttpSecurity

- Used to configure security rules

---

### 🔹 CSRF

```java
.csrf(csrf -> csrf.disable())
```

- Disable for REST APIs
- Required only for browser forms

---

### 🔹 Authorization Rules

```java
.requestMatchers("/public").permitAll()
.requestMatchers("/admin").hasRole("ADMIN")
.anyRequest().authenticated()
```

- `/public` → no login
- `/admin` → only ADMIN
- others → login required

---

## 🔥 6. Authentication Types

### ✅ httpBasic

```java
.httpBasic(Customizer.withDefaults())
```

- Used for Postman / REST APIs
- Sends credentials in header:

```
Authorization: Basic base64(username:password)
```

- Stateless

---

### ✅ formLogin

```java
.formLogin(Customizer.withDefaults())
```

- Default browser login page
- Session-based authentication

---

## ⚠️ Important

Without `httpBasic()`:

- Postman cannot authenticate
- Role-based APIs fail (401/403)

## 🔑 7. In-Memory Users

```java
@Bean
public UserDetailsService userDetailsService() {

    UserDetails user = User.withUsername("dhanush")
            .password(passwordEncoder().encode("1234"))
            .roles("USER")
            .build();

    UserDetails admin = User.withUsername("admin")
            .password(passwordEncoder().encode("admin123"))
            .roles("ADMIN")
            .build();

    return new InMemoryUserDetailsManager(user, admin);
}

@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```

---

## 🔥 8. Role-Based Access

### ✅ Correct

```java
.roles("ADMIN")
.hasRole("ADMIN")
```

### ❌ Wrong

```java
.hasRole("ROLE_ADMIN")
```

> Spring automatically adds `ROLE_` prefix internally.

## 🧱 11. Custom Security (DB-Based)

### Entity

```java
@Entity
public class User {
    @Id
    @GeneratedValue
    private Long id;

    private String username;
    private String password;
    private String role;
}
```

---

### Repository

```java
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
}
```

---

### Custom UserDetails

```java
public class CustomUserDetails implements UserDetails {

    private User user;

    public CustomUserDetails(User user) {
        this.user = user;
    }

    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        return List.of(new SimpleGrantedAuthority(user.getRole()));
    }

    @Override
    public String getPassword() {
        return user.getPassword();
    }

    @Override
    public String getUsername() {
        return user.getUsername();
    }
}
```

---

### Custom UserDetailsService

```java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Autowired
    private UserRepository repo;

    @Override
    public UserDetails loadUserByUsername(String username) {

        User user = repo.findByUsername(username)
                .orElseThrow(() -> new RuntimeException("User not found"));

        return new CustomUserDetails(user);
    }
}
```

---

### Connect to Security

```java
.userDetailsService(customUserDetailsService)
```

## 🔥 12. JWT (Modern Security)

### Flow

```
1. User → login
2. Server → validate → generate token
3. Client → send token
4. Server → validate → allow access
```

---

## 🧠 13. When to Use What?

|Scenario|Approach|
|---|---|
|Postman testing|httpBasic|
|Browser login|formLogin|
|Production APIs|JWT|
|Simple app|In-memory|
|Real apps|DB + JWT|

---

## 🎯 Final Summary

- Spring Security = Authentication + Authorization
- SecurityFilterChain = core
- httpBasic = required for APIs/Postman
- formLogin = browser login
- Roles → auto prefixed with `ROLE_`
- Custom security = DB + UserDetailsService
- JWT = stateless modern security
