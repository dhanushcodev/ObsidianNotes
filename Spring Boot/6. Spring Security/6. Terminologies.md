# 🧠 1. CSRF (Cross-Site Request Forgery)

**What it is:**  
A security attack where a malicious website tricks a logged-in user into performing unwanted actions.

### 🧠 Example:

You’re logged into your banking app.  
A malicious site sends a hidden request:

```http
POST /transfer-money
```

Browser includes cookies automatically → server thinks it's you ❌

---

### ✅ Solution:

Spring adds a **CSRF Token**

```text
Each request must include:
- CSRF Token (hidden field/header)
```

---

### 🚫 Disable CSRF when:

- Using JWT (stateless)
    
- No session/cookies
    

```java
http.csrf(csrf -> csrf.disable());
```

---

# 🌍 2. CORS (Cross-Origin Resource Sharing)

**What it is:**  
Controls which domains can access your backend.

---

### 🧠 Example:

Frontend: [http://localhost:3000](http://localhost:3000/)  
Backend: [http://localhost:8080](http://localhost:8080/)

Browser blocks request ❌ unless allowed

---

### ✅ Enable:

```java
http.cors(cors -> {});
```

Custom config:

```java
@Bean
public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(List.of("http://localhost:3000"));
    config.setAllowedMethods(List.of("GET","POST"));
    
    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    return source;
}
```

# 🔑 3. HTTP Basic Authentication

**What it is:**  
Simple authentication using username + password in every request.

---

### 🧠 Header:

```http
Authorization: Basic base64(username:password)
```

---

### ⚠️ Notes:

- Requires HTTPS
    
- Credentials sent every request
    

---

### ✅ Enable:

```java
http.httpBasic();
```

---

# 🧩 4. addFilterBefore()

**What it is:**  
Adds custom filter into Spring Security filter chain.

---

### ✅ Example:

```java
http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
```

👉 Your filter runs **before default authentication filter**

# 🧠 5. Authentication vs Authorization

|Type|Meaning|
|---|---|
|Authentication|Who are you?|
|Authorization|What can you access?|

---

### Example:

```java
http.authorizeHttpRequests(auth -> auth
    .requestMatchers("/admin").hasRole("ADMIN")
    .anyRequest().authenticated()
);
```

# 🔁 13. Session vs Stateless

### Session-based:

- Uses cookies
    
- Server stores session
    
- Needs CSRF
    

### Stateless (JWT):

- No session
    
- Token in every request
    
- CSRF not needed
    

---

# 🍪 14. Cookies

- Automatically sent by browser
    
- Used in session auth
    
- Cause CSRF risk
