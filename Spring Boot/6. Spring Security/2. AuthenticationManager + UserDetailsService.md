# 🧠 1. What is AuthenticationManager?

* `AuthenticationManager` = **main authentication engine in Spring Security**
* It verifies:

  * username
  * password

### 🔄 Flow

```
Client → Controller → AuthenticationManager → AuthenticationProvider → UserDetailsService → DB / Memory
```

# 🔧 4. Security Configuration (Spring 6)

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {

        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/login").permitAll()
                .anyRequest().authenticated()
            );

        return http.build();
    }

    // 🔥 AuthenticationManager (NEW WAY)
    @Bean
    public AuthenticationManager authenticationManager(
            AuthenticationConfiguration config) throws Exception {
        return config.getAuthenticationManager();
    }

    // Password encoder
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

# 👤 5. Hardcoded UserDetailsService

```java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Autowired
    private PasswordEncoder passwordEncoder;

    @Override
    public UserDetails loadUserByUsername(String username)
            throws UsernameNotFoundException {

        if (!username.equals("dhanush")) {
            throw new UsernameNotFoundException("User not found");
        }

        return User.builder()
                .username("dhanush")
                // ⚠️ Use pre-encoded password in real case
                .password("$2a$10$abcxyz123456789...") 
                .roles("USER")
                .build();
    }
}
```

---

# ⚠️ Password Encoding Important

❌ Wrong:

```java
passwordEncoder.encode("1234") // every request → new hash ❌
```

✅ Correct:

```java
.password("$2a$10$fixedEncodedPassword")
```

# 🎮 6. Authentication Controller

```java
@RestController
public class AuthController {

    @Autowired
    private AuthenticationManager authenticationManager;

    @PostMapping("/login")
    public ResponseEntity<?> login(@RequestBody AuthRequest request) {

        try {
            authenticationManager.authenticate(
                    new UsernamePasswordAuthenticationToken(
                            request.getUsername(),
                            request.getPassword()
                    )
            );

            return ResponseEntity.ok("Login Success");

        } catch (BadCredentialsException ex) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED)
                    .body("Invalid credentials");
        }
    }
}
```

# ❗ 10. Why You Got "BadCredentialsException"

### Log:

```
Resolved [BadCredentialsException: Bad credentials]
```

👉 Reason:

* `authenticate()` **throws exception**
* It does NOT return false

---

# ✅ 11. Fix — Handle Exception

```java
try {
    authenticationManager.authenticate(...);
} catch (BadCredentialsException ex) {
    return "Invalid credentials";
}
```

---

# 🚀 12. Better Approach — Global Exception Handler

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BadCredentialsException.class)
    public ResponseEntity<String> handleBadCredentials() {
        return ResponseEntity
                .status(HttpStatus.UNAUTHORIZED)
                .body("Invalid username or password");
    }
}
```

# ❓ 13. isAuthenticated() — Is it useful?

## 🔴 In Login API

```java
authenticationManager.authenticate(...)
```

👉 NOT needed

### Why?

* Success → returns authenticated object
* Failure → throws exception

✅ So:

```
No need to check isAuthenticated()
```

---

## ✅ Where isAuthenticated() IS used

### 1. SecurityContext

```java
Authentication auth = SecurityContextHolder.getContext().getAuthentication();

if (auth.isAuthenticated()) {
    // user is logged in
}
```

---

### 2. Filters / Middleware

```java
if (authentication != null && authentication.isAuthenticated()) {
    // allow access
}
```

---

### 3. Custom AuthenticationProvider

Used internally by Spring when building authentication object.

# ⚠️ 14. Common Mistakes

### ❌ Wrong Role Format

---

### ❌ CSRF Issues

```java
http.csrf(csrf -> csrf.disable());
```

---

### ❌ Encoding Password Every Time

Causes login failure.

# 🧾 15. Final Summary

* `AuthenticationManager.authenticate()`:

```
✔ Success → returns Authentication
❌ Failure → throws exception
```

* No need for:

```
authentication.isAuthenticated()
```

* Always:

  * Handle exceptions
  * Encode passwords properly
  * Use correct roles
