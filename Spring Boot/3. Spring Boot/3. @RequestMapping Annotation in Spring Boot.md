# @RequestMapping Annotation in Spring Boot

## 1. What is @RequestMapping?

`@RequestMapping` is an annotation used to map **HTTP requests to controller classes or methods** in Spring Boot.

It tells Spring:
- Which **URL path** should trigger a method
- Which **HTTP method** should be used
- How the request should be handled

It is mainly used inside **controller classes**.

---

## 2. Package

```java
import org.springframework.web.bind.annotation.RequestMapping;
```

---

## 3. Basic Syntax

```java
@RequestMapping(value = "/path", method = RequestMethod.GET)
```

### Attributes

| Attribute | Description |
|-----------|-------------|
| value / path | URL path to map |
| method | HTTP method (GET, POST, PUT, DELETE) |
| params | Required request parameters |
| headers | Required request headers |
| consumes | Content type the method can accept |
| produces | Content type the method returns |

---

## 4. Simple Example

```java
@RestController
public class HelloController {

    @RequestMapping("/hello")
    public String hello() {
        return "Hello World";
    }

}
```

### URL

```
http://localhost:8080/hello
```

### Output

```
Hello World
```

---

## 5. Mapping Specific HTTP Method

```java
@RestController
public class UserController {

    @RequestMapping(value = "/user", method = RequestMethod.GET)
    public String getUser() {
        return "Fetching User";
    }

}
```

Only **GET requests** are allowed.

---

## 6. Class Level Mapping

`@RequestMapping` can also be applied to the **controller class** to define a **base path**.

```java
@RestController
@RequestMapping("/api")
public class ProductController {

    @RequestMapping("/products")
    public String getProducts() {
        return "All Products";
    }

}
```

### Final URL

```
http://localhost:8080/api/products
```

---

## 7. Multiple HTTP Methods

```java
@RequestMapping(value = "/data", method = {RequestMethod.GET, RequestMethod.POST})
public String data() {
    return "Supports GET and POST";
}
```

Supports **both GET and POST requests**.

---

## 8. Consumes (Request Content Type)

Defines the **type of request body** the method accepts.

```java
@RequestMapping(value = "/users", method = RequestMethod.POST, consumes = "application/json")
public String createUser() {
    return "User created";
}
```

Accepts only **JSON requests**.

---

## 9. Produces (Response Content Type)

Defines the **response format** returned by the method.

```java
@RequestMapping(value = "/users", produces = "application/json")
public String getUsers() {
    return "{\"name\":\"John\"}";
}
```

Returns **JSON response**.

---

## 10. Shortcut Annotations (Modern Approach)

Spring provides specialized annotations instead of using `@RequestMapping`.

| Annotation | Equivalent |
|-----------|-------------|
| `@GetMapping` | `@RequestMapping(method = RequestMethod.GET)` |
| `@PostMapping` | `@RequestMapping(method = RequestMethod.POST)` |
| `@PutMapping` | `@RequestMapping(method = RequestMethod.PUT)` |
| `@DeleteMapping` | `@RequestMapping(method = RequestMethod.DELETE)` |

### Example

```java
@GetMapping("/hello")
public String hello() {
    return "Hello";
}
```

---

## 11. Real World Example

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping
    public String getUsers() {
        return "All users";
    }

    @PostMapping
    public String createUser() {
        return "User created";
    }

    @DeleteMapping("/{id}")
    public String deleteUser() {
        return "User deleted";
    }

}
```

### API Endpoints

```
GET     /users
POST    /users
DELETE  /users/{id}
```

---

## 12. Key Points

- Maps **HTTP requests to controller methods**
- Can be used at **class level and method level**
- Supports **URL mapping, HTTP methods, headers, params**
- More flexible than `@GetMapping`, `@PostMapping`, etc.
- Used in **Spring MVC controllers**
