# @RequestEntity in Spring Boot

## 1. Overview
`@RequestEntity` is used in Spring Boot to represent the **entire HTTP request** including:

- Headers
- Body
- HTTP Method
- URL

It is a wrapper around the HTTP request and is represented by the class:

```java
org.springframework.http.RequestEntity
```

It allows us to access **all request details in a single object**.

---

# 2. Why Use @RequestEntity

Normally in Spring Boot we use separate annotations:

- `@RequestHeader`
- `@RequestBody`
- `@RequestParam`
- `@PathVariable`

But `RequestEntity` allows accessing **headers and body together in one object**.

---

# 3. Basic Syntax

```java
@PostMapping("/users")
public String createUser(RequestEntity<User> requestEntity) {

    User user = requestEntity.getBody();
    HttpHeaders headers = requestEntity.getHeaders();

    return "User: " + user.getName();
}
```

---

# 4. Example

## Controller

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @PostMapping
    public String createUser(RequestEntity<User> request) {

        User user = request.getBody();
        String token = request.getHeaders().getFirst("Authorization");

        return "User: " + user.getName() + " Token: " + token;
    }
}
```

# 6. Methods Available in RequestEntity

| Method | Description |
|------|-------------|
| `getBody()` | Returns request body |
| `getHeaders()` | Returns all headers |
| `getMethod()` | Returns HTTP method |
| `getUrl()` | Returns request URL |

---

# 7. Example Accessing All Details

```java
@PostMapping("/info")
public String getRequestInfo(RequestEntity<User> request) {

    User body = request.getBody();
    HttpHeaders headers = request.getHeaders();
    HttpMethod method = request.getMethod();
    URI url = request.getUrl();

    return "Method: " + method +
           " URL: " + url +
           " Name: " + body.getName();
}
```

---

# 8. When To Use RequestEntity

Use `RequestEntity` when:

- You want **headers + body together**
- You need **full request metadata**
- Writing **generic request handlers**
- Implementing **logging, debugging, or auditing**

---

# 9. RequestEntity vs @RequestBody

| Feature | @RequestBody | RequestEntity |
|-------|-------------|---------------|
| Access Body | Yes | Yes |
| Access Headers | No | Yes |
| Access HTTP Method | No | Yes |
| Access URL | No | Yes |

---

# 10. Real World Use Case

In **API Gateway / Logging Middleware**:

```java
@PostMapping("/orders")
public ResponseEntity<String> createOrder(RequestEntity<OrderRequest> request) {

    log.info("Headers: {}", request.getHeaders());
    log.info("Body: {}", request.getBody());

    return ResponseEntity.ok("Order Created");
}
```

This helps in:

- Request auditing
- Debugging APIs
- Capturing full request information

---

# 11. Important Notes

- `RequestEntity` is part of **Spring Web module**
- It is mainly used when **entire HTTP request context is needed**
- Often used with **ResponseEntity**

`RequestEntity` is used to access the **complete HTTP request** in Spring Boot.

All inside **one object**.

# Getting PathVariable, RequestParam, Header, and Body using RequestEntity in Spring Boot

## 1. Important Concept

`RequestEntity` can directly provide:

- Request Body
- Request Headers
- HTTP Method
- Request URL

But it **cannot automatically resolve**:

- `@PathVariable`
- `@RequestParam`

Those must still be obtained either from:

- Method parameters (`@PathVariable`, `@RequestParam`)
- Parsing the URL manually from `RequestEntity.getUrl()`

---

# 2. Recommended Approach (Industry Practice)

Use **RequestEntity for body + headers**  
and **annotations for path variables and params**.

## Example Request

```
POST /users/10?active=true
Authorization: Bearer abc123
Content-Type: application/json

{
  "name": "Dhanush",
  "age": 25
}
```

---

## Controller

```java
@RestController
@RequestMapping("/users")
public class UserController {

    @PostMapping("/{id}")
    public String getRequestData(
            @PathVariable int id,
            @RequestParam boolean active,
            RequestEntity<User> request
    ) {

        // body
        User user = request.getBody();

        // headers
        String token = request.getHeaders().getFirst("Authorization");

        return "ID: " + id +
               ", Active: " + active +
               ", Token: " + token +
               ", Name: " + user.getName();
    }
}
```

---

```java
public class User {

    private String name;

        return name;

        this.name = name;
```

# 4. Getting Data Manually from RequestEntity

You can also extract query parameters manually from the URL.

```java
@PostMapping("/test")
public String test(RequestEntity<String> request) {

    URI uri = request.getUrl();

    String query = uri.getQuery(); // active=true

    HttpHeaders headers = request.getHeaders();

    String token = headers.getFirst("Authorization");

    String body = request.getBody();

    return "Query: " + query + " Token: " + token + " Body: " + body;
}
```

---

# 5. What RequestEntity Can Access

| Data | Supported |
|-----|-----------|
| Request Body | Yes |
| Headers | Yes |
| HTTP Method | Yes |
| Request URL | Yes |
| Query Params | Only by parsing URL |
| Path Variables | No |

---

# 6. Why PathVariables Are Not Available

Spring resolves path variables using **HandlerMethodArgumentResolver** before method execution.

`RequestEntity` only contains the **raw HTTP request**, not the **resolved routing variables**.

---

# 7. Best Practice

Use this combination:

```
@PathVariable  -> resource identifier
@RequestParam  -> filtering parameters
RequestEntity  -> body + headers
```

Example:

```java
@PostMapping("/orders/{orderId}")
public ResponseEntity<String> placeOrder(
        @PathVariable int orderId,
        @RequestParam String coupon,
        RequestEntity<OrderRequest> request
) {

    OrderRequest body = request.getBody();
    String authHeader = request.getHeaders().getFirst("Authorization");

    return ResponseEntity.ok("Order placed");
}
```

| Data Type     | How To Get                   |
| ------------- | ---------------------------- |
| Path Variable | `@PathVariable`              |
| Query Param   | `@RequestParam`              |
| Header        | `RequestEntity.getHeaders()` |
| Body          | `RequestEntity.getBody()`    |
| URL           | `RequestEntity.getUrl()`     |
