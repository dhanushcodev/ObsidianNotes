# @PathVariable in Spring Boot

## What is `@PathVariable`

`@PathVariable` is a Spring MVC annotation used to extract values from the **URL path** and bind them to **method parameters** in a controller.

It is commonly used in **REST APIs** to identify a specific resource.

Example URL:

```
/users/10
```

Here `10` is a **path variable**.

---

# Basic Syntax

```java
@GetMapping("/users/{id}")
public String getUser(@PathVariable int id) {
    return "User ID: " + id;
}
```

### Explanation

- `/users/{id}` → `{id}` is a **path variable**
- `@PathVariable int id` → binds the value from the URL to the method parameter

Request:

```
GET /users/10
```

Response:

```
User ID: 10
```

---

# Example Controller

```java
@RestController
@RequestMapping("/api")
public class UserController {

    @GetMapping("/users/{id}")
    public String getUser(@PathVariable int id) {
        return "User ID: " + id;
    }

}
```

Request:

```
http://localhost:8080/api/users/25
```

Response:

```
User ID: 25
```

---

# Multiple Path Variables

Spring Boot allows multiple path variables in the URL.

```java
@GetMapping("/users/{userId}/orders/{orderId}")
public String getOrder(
        @PathVariable int userId,
        @PathVariable int orderId) {

    return "User: " + userId + " Order: " + orderId;
}
```

Request:

```
GET /users/5/orders/100
```

Response:

```
User: 5 Order: 100
```

---

# Accept All Path Variables

Spring Boot allows multiple path variables in the URL.

```java
@GetMapping("/users/{userId}/orders/{orderId}")
public String getOrder(@PathVariable Map<String,String> map ) {

    return "User: " + map.get("userId") + " Order: " + map.get("orderId");
}
```

Request:

```
GET /users/5/orders/100
```

Response:

```
User: 5 Order: 100
```

---

# Explicit Variable Name Mapping

If the parameter name is different from the path variable name.

```java
@GetMapping("/users/{id}")
public String getUser(@PathVariable("id") int userId) {
    return "User ID: " + userId;
}
```

Mapping:

```
{id} → userId
```

---

# Optional Path Variable

A path variable can be optional using `required = false`.

```java
@GetMapping({"/users", "/users/{id}"})
public String getUser(@PathVariable(required = false) Integer id) {

    if (id == null) {
        return "All users";
    }

    return "User ID: " + id;
}
```

Requests:

```
/users
/users/10
```

---

# Using PathVariable with Objects

```java
@GetMapping("/products/{id}")
public Product getProduct(@PathVariable Long id) {
    return productService.getProduct(id);
}
```

Spring automatically converts the URL value to the required type.

---

# PathVariable vs RequestParam

| Feature | PathVariable | RequestParam |
|--------|-------------|--------------|
| Location | URL Path | Query Parameter |
| Example | `/users/10` | `/users?id=10` |
| Use Case | Resource identification | Filtering / optional parameters |

Example:

PathVariable

```
/users/10
```

RequestParam

```
/users?id=10
```

# Real World Example

```java
@RestController
@RequestMapping("/products")
public class ProductController {

    @GetMapping("/{id}")
    public String getProduct(@PathVariable int id) {
        return "Product ID: " + id;
    }

}
```

API Request:

```
GET /products/101
```

Response:

```
Product ID: 101
```

---

# When to Use `@PathVariable`

Use `@PathVariable` when:

- Identifying a **specific resource**
- Designing **RESTful APIs**
- The value is part of the **URL path**

Examples:

```
GET /users/10
GET /orders/200
GET /products/50
```

---

# Key Points

- Extracts values from **URL path**
- Must match `{variable}` in the URL
- Can have **multiple path variables**
- Supports **automatic type conversion**
- Commonly used in **REST APIs**

---

# Summary

`@PathVariable` is used to bind **dynamic values from the URL path** to **method parameters** in a Spring Boot controller.  
It is mainly used to **identify resources in REST APIs**.
