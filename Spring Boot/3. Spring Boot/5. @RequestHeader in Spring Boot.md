# @RequestHeader in Spring Boot

## 1. What is `@RequestHeader`?

`@RequestHeader` is a Spring MVC annotation used to **extract values from HTTP request headers** and bind them to method parameters in a controller.

It allows a Spring Boot application to read information sent in the **headers of an HTTP request**.

HTTP headers often contain metadata such as:
- Authorization tokens
- Content type
- Language preferences
- Client information

---

## 2. Basic Syntax

```java
@GetMapping("/example")
public String exampleMethod(@RequestHeader("Header-Name") String value) {
    return value;
}
```

### Explanation
- `"Header-Name"` → Name of the header from the request  
- `value` → Variable that receives the header value

---

## 3. Example

### Request

```
GET /greet
User-Agent: Chrome
```

### Controller

```java
@RestController
public class MyController {

    @GetMapping("/greet")
    public String greet(@RequestHeader("User-Agent") String userAgent) {
        return "Request came from: " + userAgent;
    }

}
```

### Output

```
Request came from: Chrome
```

---

## 4. Optional Headers

By default, `@RequestHeader` is **required**.

If the header is missing, Spring throws an exception.

You can make it optional using `required = false`.

```java
@GetMapping("/test")
public String test(@RequestHeader(value = "User-Agent", required = false) String agent) {
    return "Agent: " + agent;
}
```

If the header is missing:

```
Agent: null
```

---

## 5. Default Values

You can provide a default value if the header is not present.

```java
@GetMapping("/test")
public String test(
    @RequestHeader(value = "User-Agent", defaultValue = "Unknown") String agent
) {
    return agent;
}
```

If the header is missing:

```
Unknown
```

---

## 6. Reading All Headers

You can capture **all request headers** using a `Map`.

```java
@GetMapping("/headers")
public Map<String, String> getHeaders(@RequestHeader Map<String, String> headers) {
    return headers;
}
```

Example output:

```
{
  "host": "localhost:8080",
  "user-agent": "Chrome",
  "accept": "*/*"
}
```

## 7. Real World Use Cases

### 1. Authorization Tokens

```java
@GetMapping("/profile")
public String getProfile(@RequestHeader("Authorization") String token) {
    return "Token received: " + token;
}
```

Example Header

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
```

Used in **JWT authentication**.

---

### 2. Language Detection

```java
@GetMapping("/language")
public String language(@RequestHeader("Accept-Language") String language) {
    return "Preferred Language: " + language;
}
```

---

### 3. API Versioning

```java
@GetMapping("/api")
public String apiVersion(@RequestHeader("X-API-VERSION") String version) {
    return "API Version: " + version;
}
```

Example header

```
X-API-VERSION: v2
```

---

## 8. Important Attributes

| Attribute | Description |
|---|---|
| value | Name of the header |
| required | Whether the header must be present (default = true) |
| defaultValue | Value used if header is missing |

Example

```java
@RequestHeader(value = "User-Agent", required = false, defaultValue = "Unknown")
```

---

## 9. Difference Between `@RequestParam` and `@RequestHeader`

| Feature | @RequestParam | @RequestHeader |
|---|---|---|
| Reads data from | URL parameters | HTTP headers |
| Example | `/users?id=10` | `Authorization: Bearer token` |
| Used for | Query parameters | Request metadata |

Example

```
GET /users?id=10
```

```java
@RequestParam("id") int id
```

Header example

```
Authorization: Bearer token
```

```java
@RequestHeader("Authorization") String token
```

---

## 10. Summary

- `@RequestHeader` is used to **read HTTP headers in Spring Boot controllers**.
- It binds header values to method parameters.
- Headers can be **required, optional, or have default values**.
- Can also read **all headers using a Map**.
- Commonly used for **authentication, API versioning, and client metadata**.
