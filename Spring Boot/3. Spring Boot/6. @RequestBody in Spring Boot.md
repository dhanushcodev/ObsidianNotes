# @RequestBody in Spring Boot

## Definition
`@RequestBody` is used to bind the **HTTP request body** to a **Java object** in a Spring Boot controller.

Spring automatically:
1. Reads the request body
2. Converts JSON or XML into a Java object
3. Passes the object to the controller method

The conversion is handled by **HttpMessageConverters** (usually **Jackson** for JSON).

---

## When to Use

Use `@RequestBody` when:

- The client sends **data in the request body**
- The data format is **JSON or XML**
- You want to convert the request data into a **Java object**

Common HTTP methods:
- POST
- PUT
- PATCH

---

## Basic Example

### JSON Request

```json
{
  "name": "John",
  "age": 25
}
```

### Java Class

```java
public class User {

    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}
```

### Controller

```java
@RestController
public class UserController {

    @PostMapping("/users")
    public String createUser(@RequestBody User user) {
        return "User created: " + user.getName();
    }

}
```

---

## How It Works

1. Client sends a request with JSON in the body  
2. Spring receives the request  
3. Jackson converts JSON → Java object  
4. The object is injected into the controller method  

---

## Mapping Example

### JSON Request

```json
{
  "name": "Alice",
  "age": 30
}
```

### Controller

```java
@PostMapping("/save")
public String saveUser(@RequestBody User user) {
    return user.getName() + " is " + user.getAge() + " years old";
}
```

Spring automatically maps:

| JSON Field | Java Field |
|-------------|-------------|
| name | user.name |
| age | user.age |

---

## Using @RequestBody with Map

If you do not want to create a DTO class:

```java
@PostMapping("/data")
public String getData(@RequestBody Map<String, Object> data) {
    return data.toString();
}
```

Example JSON:

```json
{
  "city": "Bangalore",
  "temperature": 30
}
```

## Validation with @Valid

### DTO

```java
public class User {

    @NotBlank
    private String name;

    @Min(18)
    private int age;

}
```

### Controller

```java
@PostMapping("/register")
public String registerUser(@Valid @RequestBody User user) {
    return "Registered: " + user.getName();
}
```

Spring automatically validates the request body.

## @RequestBody vs @RequestParam

| Feature | @RequestBody | @RequestParam |
|--------|--------------|--------------|
| Data Source | Request Body | URL Parameters |
| Data Format | JSON / XML | Query Parameters |
| Use Case | Complex Objects | Simple Values |
| Common Methods | POST, PUT | GET |

Example:

POST request with body:

```json
{
  "name": "John"
}
```

GET request with query parameter:

```
/users?name=John
```

---

## @RequestBody vs @ModelAttribute

| Feature | @RequestBody | @ModelAttribute |
|--------|--------------|--------------|
| Data Source | Request Body | Form Data / Query Parameters |
| Data Format | JSON / XML | Form Submission |
| Usage | REST APIs | MVC Applications |

## Real World Example

### JSON Request

```json
{
  "name": "Laptop",
  "price": 70000
}
```

### Product Class

```java
public class Product {

    private String name;
    private double price;

    // getters and setters
}
```

### Controller

```java
@PostMapping("/products")
public Product createProduct(@RequestBody Product product) {
    return product;
}
```

Request:

```
POST /products
Content-Type: application/json
```

Spring automatically converts JSON → Product object.

---

## Key Points

- `@RequestBody` binds **HTTP request body → Java object**
- Works mainly with **JSON and XML**
- Uses **HttpMessageConverters** (Jackson for JSON)
- Mostly used with **POST, PUT, PATCH**
- Supports validation using **@Valid**
- Widely used in **REST APIs**
