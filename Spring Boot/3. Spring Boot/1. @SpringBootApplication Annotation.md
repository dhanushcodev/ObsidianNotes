# @SpringBootApplication Annotation

## Introduction
`@SpringBootApplication` is the main annotation used to start a Spring Boot application.  
It is usually placed on the main class of the application.

It is a combination of three important Spring annotations:

- `@Configuration`
- `@EnableAutoConfiguration`
- `@ComponentScan`

Using this single annotation enables most Spring Boot features automatically.

---

# Basic Example

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }

}
```

### What Happens Here

1. Spring Boot starts the application.
2. Spring scans components in the package.
3. Beans are automatically configured.
4. The Spring container is created and the application starts.

---

# Internal Structure

`@SpringBootApplication` is equivalent to:

```java
@Configuration
@EnableAutoConfiguration
@ComponentScan
```

---

# @Configuration

## Purpose
Marks a class as a source of bean definitions.

Spring treats the class like a configuration file.

### Example

```java
@Configuration
public class AppConfig {

    @Bean
    public Vehicle vehicle() {
        return new Vehicle();
    }

}
```

### What It Does

- Allows defining beans using `@Bean`
- Replaces old XML configuration

---

# @EnableAutoConfiguration

## Purpose
Automatically configures Spring beans based on dependencies present in the project.

Spring Boot checks:
- dependencies
- libraries
- classpath

and configures required beans automatically.

### Example

If the dependency is present:

```
spring-boot-starter-web
```

Spring Boot automatically configures:

- Embedded Tomcat server
- DispatcherServlet
- Spring MVC configuration

---

# @ComponentScan

## Purpose
Scans packages for Spring components and registers them as beans.

Spring detects classes annotated with:

- `@Component`
- `@Service`
- `@Repository`
- `@Controller`
- `@RestController`

### Example

```java
@Service
public class PaymentService {

    public void process() {
        System.out.println("Processing payment");
    }

}
```

Spring automatically creates a bean for this class.

---

# Package Scanning Rule

By default, Spring scans:

```
Current package + all sub-packages
```

Example project structure:

```
com.example.app
 ├── Application.java  (@SpringBootApplication)
 ├── service
 │     └── UserService.java
 ├── controller
 │     └── UserController.java
```

All classes inside these packages will be scanned.

---

# Customizing Component Scan

You can specify packages manually.

```java
@SpringBootApplication(scanBasePackages = "com.example")
public class MyApplication {
}
```

Or

```java
@ComponentScan("com.example.service")
```

---

# Excluding Auto Configuration

Sometimes certain auto configurations need to be disabled.

Example:

```java
@SpringBootApplication(exclude = DataSourceAutoConfiguration.class)
public class MyApplication {
}
```

Use case:

- Application does not use a database
- Prevent unnecessary configuration

---

# Application Startup Flow

1. `SpringApplication.run()` starts the Spring Boot application.
2. `@SpringBootApplication` triggers configuration.
3. Component scanning registers beans.
4. Auto configuration configures required components.
5. Embedded server (Tomcat/Jetty) starts.
6. Application becomes ready.

# Simple Real World Example

### Controller

```java
@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello Spring Boot";
    }

}
```

### Main Class

```java
@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }

}
```

### Access URL

```
http://localhost:8080/hello
```

Response:

```
Hello Spring Boot
```

# Advantages

- Reduces configuration
- Enables automatic configuration
- Simplifies project setup
- Automatically scans components
- Combines multiple annotations into one

---

# Summary

| Annotation | Purpose |
|---|---|
| `@SpringBootApplication` | Main annotation to start Spring Boot |
| `@Configuration` | Defines bean configuration |
| `@EnableAutoConfiguration` | Automatically configures beans |
| `@ComponentScan` | Scans and registers components |
