# @RequestParam in Spring Boot

## 1. Definition
`@RequestParam` is used to bind **HTTP request parameters (query parameters or form data)** to method parameters in a Spring Boot controller.

It extracts values from the **URL query string** and assigns them to method arguments.

Example URL:
```
http://localhost:8080/greet?name=Dhanush
```

---

# 2. Basic Syntax

```java
@GetMapping("/endpoint")
public String method(@RequestParam Type variableName) {
}
```

---

# 3. Basic Example

### URL
```
http://localhost:8080/greet?name=Dhanush
```

### Controller

```java
@RestController
public class MyController {

    @GetMapping("/greet")
    public String greet(@RequestParam String name) {
        return "Hello " + name;
    }

}
```

### Output
```
Hello Dhanush
```

Spring extracts `name` from the URL and assigns it to the method parameter.

---

# 4. Custom Parameter Name

If the method parameter name is different from the request parameter name.

### URL
```
http://localhost:8080/greet?username=Dhanush
```

```java
@GetMapping("/greet")
public String greet(@RequestParam("username") String name) {
    return "Hello " + name;
}
```

`username` from the URL is mapped to the variable `name`.

---

# 5. Required Parameter (Default Behavior)

By default, `@RequestParam` is **required = true**.

If the parameter is missing, Spring throws an exception.

```java
@GetMapping("/hello")
public String hello(@RequestParam String name) {
    return "Hello " + name;
}
```

### Request
```
/hello
```

### Result
```
400 Bad Request
Required request parameter 'name' is not present
```

---

# 6. Optional Parameter

Set `required = false`.

```java
@GetMapping("/hello")
public String hello(@RequestParam(required = false) String name) {
    return "Hello " + name;
}
```

### Requests

```
/hello?name=Dhanush
```

Output:
```
Hello Dhanush
```

```
/hello
```

Output:
```
Hello null
```

---

# 7. Default Value

You can provide a default value if the parameter is missing.

```java
@GetMapping("/hello")
public String hello(@RequestParam(defaultValue = "Guest") String name) {
    return "Hello " + name;
}
```

### Requests

```
/hello?name=Dhanush
```

Output:
```
Hello Dhanush
```

```
/hello
```

Output:
```
Hello Guest
```

---

# 8. Multiple Request Parameters

Example URL:
```
http://localhost:8080/add?a=5&b=10
```

```java
@GetMapping("/add")
public int add(@RequestParam int a, @RequestParam int b) {
    return a + b;
}
```

Output:
```
15
```

---

# 9. Using @RequestParam with Collections

Spring can bind multiple values to a list.

### URL
```
http://localhost:8080/fruits?name=apple&name=mango&name=banana
```

```java
@GetMapping("/fruits")
public List<String> getFruits(@RequestParam List<String> name) {
    return name;
}
```

Output
```
[apple, mango, banana]
```

---

# 10. Map Binding

Capture all request parameters into a map.

```java
@GetMapping("/params")
public Map<String, String> getParams(@RequestParam Map<String, String> params) {
    return params;
}
```

### URL
```
/params?name=Dhanush&age=22
```

Output
```
{name=Dhanush, age=22}
```

---

# 11. @RequestParam vs @PathVariable

| Feature | @RequestParam | @PathVariable |
|--------|---------------|---------------|
| Data Source | Query parameters | URL path |
| Example | `/users?id=10` | `/users/10` |
| Optional Support | Yes | Usually required |
| Use Case | Filtering / optional inputs | Resource identification |

Example:

```
/users?id=10       -> @RequestParam
/users/10          -> @PathVariable
```

# 12. Real World Use Cases

### Pagination

```
/products?page=1&size=10
```

```java
@GetMapping("/products")
public List<Product> getProducts(
        @RequestParam int page,
        @RequestParam int size) {

    return service.getProducts(page, size);
}
```

---

### Search API

```
/search?keyword=laptop
```

```java
@GetMapping("/search")
public List<Product> search(
        @RequestParam String keyword) {

    return service.search(keyword);
}
```

---

# 13. Key Points

- Used to read **query parameters from HTTP requests**
- Works with **GET and POST requests**
- Default behavior is **required = true**
- Can set **default values**
- Supports **List, Map, and multiple parameters**
- Commonly used for **search, filtering, pagination**
