# 🔹 H2 Database in Spring Boot

## 🔸 What is H2?

- Lightweight **in-memory relational database**
- Written in Java
- No installation required
- Mostly used for **development & testing**

---

## 🔸 Key Features

- In-memory (data lost on restart)
- Fast and lightweight
- Embedded database
- Supports SQL
- Provides **web console UI**

---

## 🔸 H2 Modes

### 1. In-Memory (Most Used)
```
jdbc:h2:mem:testdb
```
- Data is temporary (lost after restart)

### 2. File-Based
```
spring.datasource.url=jdbc:h2:file:./data/testdb
spring.sql.init.mode=always
```
- Data is stored in file (persistent)

---

## 🔸 Dependency (Maven)

```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

---

## 🔸 Configuration (application.properties)

```properties
# Enable H2 Console
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console

# Datasource
spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.driverClassName=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# JPA
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

| Property  | Value |
|----------|------|

## 🔸 Example (Spring Boot + H2)

### Entity
```java
@Entity
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private String email;
}
```

---

### Repository
```java
public interface UserRepository extends JpaRepository<User, Long> {
}
```

---

### Controller
```java
@RestController
@RequestMapping("/users")
public class UserController {

    private final UserRepository repo;

    public UserController(UserRepository repo) {
        this.repo = repo;
    }

    @PostMapping
    public User save(@RequestBody User user) {
        return repo.save(user);
    }

    @GetMapping
    public List<User> getAll() {
        return repo.findAll();
    }
}
```

---

## 🔸 When to Use H2

### ✅ Use
- Development
- Testing
- Learning JPA

### ❌ Avoid
- Production
- Large datasets
- Persistent storage

# 🔹 schema.sql & data.sql in Spring Boot

---

## 🔸 What are schema.sql and data.sql?

| File        | Purpose |
|------------|--------|
| `schema.sql` | Defines **database structure (DDL)** |
| `data.sql`   | Inserts **initial data (DML)** |

---

## 🔸 Execution Order

1. `schema.sql` → creates tables  
2. `data.sql` → inserts data  

---

## 🔸 File Location

```
src/main/resources/
    ├── schema.sql
    └── data.sql
```

---

## 🔸 schema.sql (DDL – Structure)

### Example
```sql
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(150) UNIQUE
);
```

---

## 🔸 data.sql (DML – Data)

### Example
```sql
INSERT INTO users(name, email) VALUES ('Alice', 'alice@gmail.com');
INSERT INTO users(name, email) VALUES ('Bob', 'bob@gmail.com');
```

---

## 🔸 Default Behavior

- Auto-runs for embedded DBs (H2, HSQL, Derby)

---

## 🔸 Force Execution

```properties
spring.sql.init.mode=always
```

---

## 🔸 Hibernate Interaction ⚠️

### Option 1: Hibernate
```properties
spring.jpa.hibernate.ddl-auto=update
```

### Option 2: schema.sql
```properties
spring.jpa.hibernate.ddl-auto=none
```

❗ Do not use both together

---

## 🔸 Common Issue

### Problem
`data.sql` runs before tables are created

### Solution
```properties
spring.jpa.defer-datasource-initialization=true
```

👉 Ensures:
Hibernate creates tables → THEN `data.sql` runs

---

## 🔸 Advanced Configuration

```properties
spring.sql.init.mode=always
spring.sql.init.schema-locations=classpath:my-schema.sql
spring.sql.init.data-locations=classpath:my-data.sql
spring.sql.init.continue-on-error=true
```

---

## 🔸 Multiple Files

```
schema-users.sql
schema-orders.sql
data-users.sql
data-orders.sql
```

```properties
spring.sql.init.schema-locations=classpath:schema-*.sql
spring.sql.init.data-locations=classpath:data-*.sql
```

---

## 🔸 Best Practices

- Use Hibernate for development
- Use schema.sql for production control
- Keep data.sql minimal
- Avoid mixing schema.sql + Hibernate
- Maintain proper execution order

---

# 🔥 Final Summary

- H2 = lightweight in-memory DB for dev/testing
- schema.sql = structure (DDL)
- data.sql = data (DML)
- Runs automatically with H2
- Order matters (schema → data)
- Use `defer-datasource-initialization` to fix timing issues
- Avoid conflicts with Hibernate
