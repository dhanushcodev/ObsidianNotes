# Naming the Beans in Spring / Spring Boot

In **Spring / Spring Boot**, every object managed by the Spring container is called a **Bean**, and each bean must have a **unique name (Bean ID)** inside the **ApplicationContext**.

Bean naming is important because Spring uses the **bean name to retrieve the object from the container**.

---

# 1. Default Bean Naming (Automatic)

If you **do not specify a name**, Spring automatically generates one.

### Rule
Spring uses the **class name with the first letter in lowercase**.

### Example

```java
@Component
public class UserService {
}
```

Bean name generated by Spring:

```
userService
```

Usage:

```java
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

UserService service = context.getBean(UserService.class);
```

or

```java
UserService service = (UserService) context.getBean("userService");
```

---

# 2. Naming Bean Using `@Component("name")`

You can explicitly provide a bean name.

### Example

```java
@Component("myUserService")
public class UserService {
}
```

Bean name:

```
myUserService
```

Usage:

```java
UserService service = (UserService) context.getBean("myUserService");
```

---

# 3. Naming Beans Using `@Bean` Annotation

When creating beans in **Java Configuration**, the method name becomes the **bean name** by default.

### Example

```java
@Configuration
public class AppConfig {

    @Bean
    public UserService userService() {
        return new UserService();
    }
}
```

Bean name:

```
userService
```

Usage:

```java
UserService service = context.getBean("userService", UserService.class);
```

---

# 4. Custom Bean Name Using `@Bean(name="...")`

You can give a custom name.

### Example

```java
@Configuration
public class AppConfig {

    @Bean(name = "customUserService")
    public UserService createService() {
        return new UserService();
    }
}
```

Bean name:

```
customUserService
```

Usage:

```java
UserService service = context.getBean("customUserService", UserService.class);
```

---

# 5. Multiple Names (Aliases) for a Bean

You can give **multiple names to the same bean**.

### Example

```java
@Bean(name = {"userService", "serviceBean"})
public UserService createService() {
    return new UserService();
}
```

Both will work:

```java
context.getBean("userService");
context.getBean("serviceBean");
```

Both refer to the **same bean object**.

---

# 6. Naming Beans with Stereotype Annotations

Spring stereotypes also support bean naming.

### `@Service`

```java
@Service("paymentService")
public class PaymentService {
}
```

### `@Repository`

```java
@Repository("userRepository")
public class UserRepository {
}
```

### `@Controller`

```java
@Controller("homeController")
public class HomeController {
}
```

---

# 7. Bean Naming When Using `getBean()`

When retrieving beans using **getBean**, the bean name matters.

### Example

```java
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

UserService service = (UserService) context.getBean("userService");
```

If the name is wrong:

```
NoSuchBeanDefinitionException
```

---

# 8. Bean Naming Conflict

If two beans have the **same name**, Spring throws:

```
ConflictingBeanDefinitionException
```

Example:

```java
@Component("userService")
class A {}

@Component("userService")
class B {}
```

Error because **bean names must be unique**.

---

# 9. Best Practice for Bean Naming

Use **camelCase naming**.

Examples:

```
userService
orderService
paymentGateway
emailSender
```

Avoid:

```
UserService
service1
beanObject
```

---

# Quick Summary

| Method | Bean Name |
|------|------|
| `@Component` | class name (camelCase) |
| `@Component("name")` | custom name |
| `@Bean` | method name |
| `@Bean(name="...")` | custom name |
| `@Service("name")` | custom name |
| `@Repository("name")` | custom name |

#### Practice Code

ProjectConfig.java

```java
package com.spring.demo.config;

import com.spring.demo.beans.Vehicle;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Description;

@Configuration
public class ProjectConfig {

    @Bean
    Vehicle vehicle1(){
        Vehicle vehicle = new Vehicle();
        vehicle.setName("David putar");
        return vehicle;
    }

    @Bean("secondVehicle")
    @Description("Vehicle 2")
    Vehicle vehicle2(){
        Vehicle vehicle = new Vehicle();
        vehicle.setName("David putar");
        return vehicle;
    }

    @Bean
    String hello(){
        return "Hello world";
    }
}
```

Main.java

```java
import com.spring.demo.beans.Vehicle;
import com.spring.demo.config.ProjectConfig;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

void main(){

    var context = new AnnotationConfigApplicationContext(ProjectConfig.class);

    Vehicle veh = (Vehicle) context.getBean("vehicle1");
    System.out.println(veh.getName());

    Vehicle veh2 = (Vehicle) context.getBean("secondVehicle");
    System.out.println(veh2.getName());

}

```
