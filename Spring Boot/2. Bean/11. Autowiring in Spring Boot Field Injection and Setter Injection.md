# Autowiring in Spring Boot

Autowiring is a feature of Spring that automatically injects required dependencies into a class without manually creating objects.

Instead of creating objects using `new`, Spring automatically finds the required bean from the **Spring IoC Container** and injects it.

Autowiring is done using:

```java
@Autowired
```

---

# Example Scenario

We have:

- `Engine` → Dependency  
- `Car` → Dependent Class  

Spring automatically injects the `Engine` bean into `Car`.

---

# 1. Field Injection

## Definition

Field Injection injects the dependency **directly into the class field** using `@Autowired`.

---

## Example

### Engine Class

```java
import org.springframework.stereotype.Component;

@Component
public class Engine {

    public void start() {
        System.out.println("Engine started");
    }
}
```

### Car Class

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class Car {

    @Autowired
    private Engine engine;

    public void drive() {
        engine.start();
        System.out.println("Car is driving");
    }
}
```

### Main Class

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {

        ApplicationContext context =
                SpringApplication.run(DemoApplication.class, args);

        Car car = context.getBean(Car.class);
        car.drive();
    }
}
```

---

## How Field Injection Works

1. Spring creates the `Engine` bean.  
2. Spring creates the `Car` bean.  
3. Spring detects `@Autowired` on the field.  
4. Spring injects the `Engine` bean into the `Car` object.

---

## Advantages of Field Injection

- Less boilerplate code  
- Simple and quick to implement  
- Easy for small projects  

---

## Disadvantages of Field Injection

- Field cannot be set to final
- Hard to test (difficult to mock dependencies)  
- Breaks encapsulation because private fields are injected using reflection  
- Dependency is hidden  
- Not recommended by the Spring team  

---

# 2. Setter Injection

## Definition

Setter Injection injects the dependency through a **setter method** annotated with `@Autowired`.

---

## Example

### Engine Class

```java
import org.springframework.stereotype.Component;

@Component
public class Engine {

    public void start() {
        System.out.println("Engine started");
    }
}
```

---

### Car Class

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class Car {

    private Engine engine;

    @Autowired
    public void setEngine(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
        System.out.println("Car is driving");
    }
}
```

---

### Main Class

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {

        ApplicationContext context =
                SpringApplication.run(DemoApplication.class, args);

        Car car = context.getBean(Car.class);
        car.drive();
    }
}
```

---

## How Setter Injection Works

1. Spring creates the `Engine` bean.  
2. Spring creates the `Car` bean.  
3. Spring detects the setter method annotated with `@Autowired`.  
4. Spring calls the setter method and injects the dependency.

---

## Advantages of Setter Injection

- Flexible because dependency can be changed later  
- Good for optional dependencies  
- Easier to modify or reconfigure  

---

## Disadvantages of Setter Injection

- Dependency is not mandatory  
- Object can exist without the dependency  
- More boilerplate code than field injection  
- Object may be incomplete before setter method runs  

---

# Field Injection vs Setter Injection

| Feature | Field Injection | Setter Injection |
|--------|----------------|-----------------|
| Injection Location | Directly on field | Through setter method |
| Boilerplate Code | Less | More |
| Testability | Hard | Easier |
| Dependency Mandatory | Yes but hidden | No |
| Encapsulation | Breaks encapsulation | Maintains encapsulation |
| Recommended | Not recommended | Acceptable |

---

# Important Note

Modern Spring development recommends **Constructor Injection instead of both**, because it:

- Makes dependencies mandatory  
- Improves testability  
- Ensures immutability  


### Practice code:

Vehicle.java

```java
package com.spring.demo.beans;

import org.springframework.stereotype.Component;

@Component
public class Vehicle {
    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Vehicle{" +
                "name='" + name + '\'' +
                '}';
    }
}

```

Person.java

```java
package com.spring.demo.beans;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class Person {

    private String name;
    private Vehicle vehicle;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public Vehicle getVehicle() {
        return vehicle;
    }

    @Autowired
    public void setVehicle(Vehicle vehicle) {
        this.vehicle = vehicle;
    }
}

```

AppConfig.java

```java
package com.spring.demo.config;  
  
import org.springframework.context.annotation.ComponentScan;  
import org.springframework.context.annotation.Configuration;  
  
@Configuration  
@ComponentScan("com.spring.demo.beans")  
public class AppConfig {  
      
}

```

Main.java:

```java

import com.spring.demo.beans.Person;  
import com.spring.demo.config.AppConfig;  
import org.springframework.context.annotation.AnnotationConfigApplicationContext;  
  
void main(){  
  
    var context = new AnnotationConfigApplicationContext(AppConfig.class);  
  
    Person per = context.getBean(Person.class);  
    per.setName("test person");  
    per.getVehicle().setName("test vehicle");  
    System.out.println(per.getName()+" "+per.getVehicle());  
}

```
