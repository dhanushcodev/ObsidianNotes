# Bean Scopes in Spring Boot

## What is Bean Scope?

Bean Scope defines **how many instances of a bean are created and how long they live in the Spring container**.

It determines the **lifecycle and visibility** of a bean.

---

# Types of Bean Scopes in Spring

## 1. Singleton Scope (Default)

### Definition
Only **one instance of the bean is created per Spring IoC container**.  
Every request for that bean returns the **same instance**.

### Annotation

```java
@Scope("singleton") //or //@Scope(BeanDefinition.SCOPE_PROTOTYPE)
```

### Example

```java
@Component
@Scope("singleton")
public class Car {
}
```

```java
@Autowired
Car car1;

@Autowired
Car car2;
```

### Result

```
car1 == car2 → true
```

Both references point to the **same object**.

### Characteristics

- Default scope in Spring
- Only one instance per container
- Created when the container starts
- Shared across the application

### Real Use Cases

- Service classes
- Repository classes
- Utility classes

Example:

```
UserService
OrderService
PaymentService
```

These classes are typically **singleton**.

---

# 2. Prototype Scope

### Definition

A **new bean instance is created every time the bean is requested**.

### Annotation

```java
@Scope("prototype")
```

### Example

```java
@Component
@Scope("prototype")
public class Car {
}
```

```java
Car car1 = context.getBean(Car.class);
Car car2 = context.getBean(Car.class);
```

### Result

```
car1 == car2 → false
```

Each request returns a **new instance**.

### Characteristics

- Multiple instances
- Created only when requested
- Spring **does not manage destruction**

### Real Use Cases

Objects that maintain **temporary state**.

Example:

```
ShoppingCart
TemporaryReport
UserRequestData
```

---

# 3. Request Scope (Web Applications)

### Definition

A **new bean instance is created for each HTTP request**.

### Annotation

```java
@Scope(value = WebApplicationContext.SCOPE_REQUEST)
```

### Example

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_REQUEST)
public class RequestData {
}
```

### Behavior

```
HTTP Request 1 → new bean
HTTP Request 2 → new bean
```

Each request gets its **own bean instance**.

### Real Use Cases

- Request-specific data
- Request logging
- Request context storage

---

# 4. Session Scope (Web Applications)

### Definition

One bean instance is created **per HTTP session**.

Each user session gets its **own bean instance**.

### Annotation

```java
@Scope(value = WebApplicationContext.SCOPE_SESSION)
```

### Example

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_SESSION)
public class UserSession {
}
```

### Behavior

```
User A session → Bean A
User B session → Bean B
```

### Real Use Cases

- Shopping cart
- Logged-in user data
- User preferences

---

# 5. Application Scope (Web Applications)

### Definition

One bean instance **per ServletContext for the entire web application**.

### Annotation

```java
@Scope(value = WebApplicationContext.SCOPE_APPLICATION)
```

### Example

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_APPLICATION)
public class AppConfigData {
}
```

### Real Use Cases

- Global configuration
- Application-wide cache
- Shared metadata

---

# 6. WebSocket Scope

### Definition

One bean instance **per WebSocket session**.

### Annotation

```java
@Scope("websocket")
```

### Example

```java
@Component
@Scope("websocket")
public class WebSocketSessionBean {
}
```

### Real Use Cases

- Chat applications
- Real-time messaging systems
- WebSocket user state

---

# Summary Table

| Scope | Instances Created | Lifetime | Typical Use |
|------|------|------|------|
| Singleton | One per container | Application lifetime | Services, repositories |
| Prototype | New every request | Until garbage collected | Stateful objects |
| Request | One per HTTP request | Request lifecycle | Request data |
| Session | One per user session | Session lifecycle | User data |
| Application | One per web application | App lifecycle | Global shared data |
| WebSocket | One per WebSocket session | WebSocket lifecycle | Real-time sessions |

# Important Concept

## Singleton Bean Injecting Prototype Bean

If a **singleton bean injects a prototype bean**, Spring creates the prototype **only once during injection**.

Example:

```java
@Component
public class CarService {

    @Autowired
    private Engine engine;
}
```

```java
@Component
@Scope("prototype")
public class Engine {
}
```

Even though `Engine` is prototype, **only one instance is injected**.

### Solution

Use `ObjectProvider`.

```java
@Autowired
private ObjectProvider<Engine> engineProvider;

public void start() {
    Engine engine = engineProvider.getObject();
}
```

Now a **new prototype bean is created each time**.

---

# Key Points

- **Singleton is the default scope**
- **Prototype creates a new instance every time**
- **Request, Session, Application scopes are used in web applications**
- Singleton beans are ideal for **stateless services**
- Prototype beans are useful for **stateful objects**
