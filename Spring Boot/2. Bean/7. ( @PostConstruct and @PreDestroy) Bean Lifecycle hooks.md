
## `@PostConstruct` and `@PreDestroy` in Spring

These annotations are used to execute **custom initialization and cleanup logic** in Spring-managed beans.

They belong to the **JSR-250 lifecycle annotations** and are executed automatically by the Spring container.

Include dependency 

```xml
<dependency>  
    <groupId>jakarta.annotation</groupId>  
    <artifactId>jakarta.annotation-api</artifactId>  
    <version>3.0.0</version>  
</dependency>
```
---

# 1. `@PostConstruct`

## Definition
`@PostConstruct` is used to **execute a method immediately after the bean is created and dependencies are injected**.

It runs **once after the bean initialization phase**.

### Lifecycle Position

```
Bean Created → Dependencies Injected → @PostConstruct → Bean Ready
```

### Rules

- Method must **not have parameters**
- Method must **return void**
- Should be used for **initialization logic**

---

## Example

```java
import jakarta.annotation.PostConstruct;
import org.springframework.stereotype.Component;

@Component
public class DatabaseConnection {

    public DatabaseConnection() {
        System.out.println("Constructor called");
    }

    @PostConstruct
    public void init() {
        System.out.println("Database Connection Initialized");
    }
}
```

### Output

```
Constructor called
Database Connection Initialized
```

### Explanation

1. Spring creates the bean.
2. Constructor executes.
3. Spring injects dependencies.
4. Method annotated with `@PostConstruct` executes.

---

## Real World Example

Example: **Loading configuration or establishing a database connection**

```java
@Component
public class CacheLoader {

    @PostConstruct
    public void loadCache() {
        System.out.println("Loading data into cache...");
    }
}
```

Used for:

- Loading cache
- Opening DB connections
- Initializing resources
- Reading configuration files

---

# 2. `@PreDestroy`

## Definition

`@PreDestroy` is used to **execute a method just before the bean is destroyed**.

This is useful for **cleanup logic**.

### Lifecycle Position

```
Bean Ready → Application Shutdown → @PreDestroy → Bean Destroyed
```

### Rules

- Method must **not take parameters**
- Method must **return void**

---

## Example

```java
import jakarta.annotation.PreDestroy;
import org.springframework.stereotype.Component;

@Component
public class DatabaseConnection {

    @PreDestroy
    public void closeConnection() {
        System.out.println("Closing Database Connection");
    }
}
```

### Output (When Application Stops)

```
Closing Database Connection
```

---

## Real World Example

```java
@Component
public class FileService {

    @PreDestroy
    public void closeFile() {
        System.out.println("Closing file resources...");
    }
}
```

Used for:

- Closing DB connections
- Closing files
- Releasing resources
- Stopping background threads

---

# Complete Example (Both Together)

```java
import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.springframework.stereotype.Component;

@Component
public class UserService {

    public UserService() {
        System.out.println("Constructor called");
    }

    @PostConstruct
    public void init() {
        System.out.println("UserService Bean Initialized");
    }

    @PreDestroy
    public void destroy() {
        System.out.println("UserService Bean Destroyed");
    }
}
```

### Output

When Application Starts

```
Constructor called
UserService Bean Initialized
```

When Application Stops

```
UserService Bean Destroyed
```

---

# Bean Lifecycle with These Annotations

```
1. Spring Container Starts
2. Bean Created
3. Dependencies Injected
4. @PostConstruct Executes
5. Bean Ready to Use
6. Application Running
7. Application Stops
8. @PreDestroy Executes
9. Bean Destroyed
```

---

# When To Use Them

| Annotation | Use Case |
|---|---|
| `@PostConstruct` | Initialization logic |
| `@PreDestroy` | Cleanup logic |

---

# Alternative Ways (Without These Annotations)

Spring also provides:

- `InitializingBean` → `afterPropertiesSet()`
- `DisposableBean` → `destroy()`
- `@Bean(initMethod="", destroyMethod="")`

Example:

```java
@Bean(initMethod = "init", destroyMethod = "cleanup")
public MyService myService() {
    return new MyService();
}
```

---

# Important Interview Point

`@PreDestroy` **will NOT run for prototype beans**.

Reason:  
Spring **does not manage the full lifecycle of prototype beans**.

It only creates them.

---

# Summary

| Annotation | Purpose | Runs When |
|---|---|---|
| `@PostConstruct` | Initialization logic | After dependency injection |
| `@PreDestroy` | Cleanup logic | Before bean destruction |
