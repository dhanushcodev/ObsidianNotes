
# Using Multiple Configuration Classes in `AnnotationConfigApplicationContext` and `@Import` in Spring


# 1. Using Multiple Configuration Classes in `AnnotationConfigApplicationContext`

In **Spring Core**, we can create the application context using **multiple configuration classes**.  
Each configuration class defines beans using `@Bean`.

Spring will combine all these configurations and manage the beans together.

---

## Example: Multiple Configuration Classes

### 1️⃣ First Configuration Class

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig1 {

    @Bean
    public String message() {
        return "Hello from AppConfig1";
    }
}
```

---

### 2️⃣ Second Configuration Class

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig2 {

    @Bean
    public Integer number() {
        return 100;
    }
}
```

---

### 3️⃣ Main Class Using Multiple Configurations

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {

    public static void main(String[] args) {

        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig1.class, AppConfig2.class);

        String msg = context.getBean(String.class);
        Integer num = context.getBean(Integer.class);

        System.out.println(msg);
        System.out.println(num);
    }
}
```

---

## What Happens Internally

1. Spring loads **AppConfig1**
2. Spring loads **AppConfig2**
3. All beans from both classes are registered in the **same IoC container**
4. Beans can be accessed normally

---

## Output

```
Hello from AppConfig1
100
```

---

## Alternative Way (Register Method)

Instead of passing classes in the constructor, we can register them manually.

```java
AnnotationConfigApplicationContext context =
        new AnnotationConfigApplicationContext();

context.register(AppConfig1.class);
context.register(AppConfig2.class);

context.refresh();
```

# 2. Using `@Import` in Spring

`@Import` allows **one configuration class to include another configuration class**.

This is useful for **modular configuration**.

---

## Example Without `@Import`

```java
new AnnotationConfigApplicationContext(AppConfig1.class, AppConfig2.class);
```

Both configs must be passed manually.

---

## Example With `@Import`

### AppConfig1

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Bean;

@Configuration
public class AppConfig1 {

    @Bean
    public String message() {
        return "Hello from Config1";
    }
}
```

---

### AppConfig2 Importing AppConfig1

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Import;

@Configuration
@Import(AppConfig1.class)
public class AppConfig2 {

}
```

---

### Main Class

Now only **AppConfig2** is needed.

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {

    public static void main(String[] args) {

        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig2.class);

        String msg = context.getBean(String.class);
        System.out.println(msg);
    }
}
```

---

## Output

```
Hello from Config1
```

---

# 3. Real World Example

In large applications we separate configuration:

```
DatabaseConfig
SecurityConfig
WebConfig
ServiceConfig
```

Then import them into a **main configuration class**.

---

## Example

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Import;

@Configuration
@Import({
        DatabaseConfig.class,
        SecurityConfig.class,
        ServiceConfig.class
})
public class AppConfig {
}
```

Then start Spring using only:

```java
AnnotationConfigApplicationContext context =
        new AnnotationConfigApplicationContext(AppConfig.class);
```

---

# 4. When to Use `@Import`

Use `@Import` when:

- You want **modular configuration**
- You want **clean architecture**
- You want **one main config class**
- Managing **large applications**

---

# 5. `@Import` Can Import 3 Things

## 1️⃣ Configuration Classes

```java
@Import(DatabaseConfig.class)
```

---

## 2️⃣ Regular Classes

```java
@Import(MyService.class)
```

Spring will create the bean automatically.

---

## 3️⃣ `ImportSelector` or `ImportBeanDefinitionRegistrar`

Used for **advanced framework development** (like Spring Boot auto configuration).

---

# 6. Quick Comparison

| Method | Usage |
|------|------|
| Multiple configs in context | `new AnnotationConfigApplicationContext(A.class, B.class)` |
| `@Import` | One config loads other configs |
| Register method | `context.register(A.class)` |

---

# 7. Interview Tip

### Question
Why use `@Import` instead of passing multiple classes to `AnnotationConfigApplicationContext`?

### Answer

- Improves **modular configuration**
- Makes **main configuration cleaner**
- Used heavily in **Spring Boot Auto Configuration**
- Enables **layered architecture**

---

## Bonus Concept (Important for Interviews)

Spring Boot internally uses **`@Import`** for **Auto Configuration** through:

```
@EnableAutoConfiguration
```

This loads many internal configuration classes automatically.

#### Practice code:

ProjectConfig.java

```java
package com.spring.demo.config;

import com.spring.demo.beans.Vehicle;
import org.springframework.context.annotation.*;

@Configuration
@Import({AnotherProjectConfig.class})
public class ProjectConfig {

    @Bean
    @Primary
    Vehicle vehicle1(){
        Vehicle vehicle = new Vehicle();
        vehicle.setName("David putar");
        return vehicle;
    }

    @Bean("secondVehicle")
    @Description("Vehicle 2")
    Vehicle vehicle2(){
        Vehicle vehicle = new Vehicle();
        vehicle.setName("David putar");
        return vehicle;
    }

    @Bean
    String hello(){
        return "Hello world";
    }
}
```

AnotheProjectConfig.java

```java
package com.spring.demo.config;

import com.spring.demo.beans.Vehicle;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AnotherProjectConfig {

    @Bean
    Vehicle anotherVehicle(){
        Vehicle vel = new Vehicle();
        vel.setName("Another vehicle");
        return vel;
    }
}
```

Main.java

```java
import com.spring.demo.beans.Vehicle;
import com.spring.demo.config.ProjectConfig;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

void main(){

    var context = new AnnotationConfigApplicationContext(ProjectConfig.class);
    
    Vehicle veh = (Vehicle) context.getBean("vehicle1");
    System.out.println(veh.getName());

    Vehicle another = (Vehicle) context.getBean("anotherVehicle");
    System.out.println(another.getName());

    Vehicle veh2 = (Vehicle) context.getBean("secondVehicle");
    System.out.println(veh2.getName());
}

```
