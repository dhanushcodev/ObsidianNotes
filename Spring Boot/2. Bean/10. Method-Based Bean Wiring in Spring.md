# Method-Based Bean Wiring in Spring

## 1. What is Method-Based Bean Wiring?

In Spring / Spring Boot, **method-based bean wiring** means creating and connecting beans using **methods inside a configuration class**.

Instead of automatic detection using `@Component`, beans are **explicitly created using `@Bean` methods** inside a `@Configuration` class.

Spring automatically **injects dependencies through method parameters**.

### Used When
- Full control over bean creation is required
- Configuring **third-party classes**
- Explicit dependency wiring is needed

---

# 2. Basic Example

## Step 1 — Create Classes

```java
public class Engine {

    public void start() {
        System.out.println("Engine started");
    }
}
```

```java
public class Car {

    private Engine engine;

    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
        System.out.println("Car is driving");
    }
}
```

---

## Step 2 — Configuration Class

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {

    @Bean
    public Engine engine() {
        return new Engine();
    }

    @Bean
    public Car car(Engine engine) {
        return new Car(engine);
    }
}
```

---

# 3. How Dependency Injection Works

Example method:

```java
@Bean
public Car car(Engine engine) {
    return new Car(engine);
}
```

Spring detects:

```
Car needs Engine
```

Flow:

```
Spring Container
        │
        ▼
engine()  → creates Engine bean
        │
        ▼
car(Engine engine) → Spring injects Engine
        │
        ▼
Car bean created
```

Spring automatically resolves dependencies **from the Application Context**.

---

# 4. Running the Application

```java
import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {

    public static void main(String[] args) {

        ApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        Car car = context.getBean(Car.class);
        car.drive();
    }
}
```

Output

```
Engine started
Car is driving
```

---

# 5. Why It Is Called Method-Based Wiring

Dependencies are injected through **method parameters**.

Example:

```java
@Bean
public Car car(Engine engine)
```

Here the **method parameter performs dependency injection**.

---

# 6. Method-Based Wiring vs Component Scanning

| Feature | Method-Based (`@Bean`) | Component-Based (`@Component`) |
|------|------|------|
| Bean creation | Explicit | Automatic |
| Configuration | Java method | Class annotation |
| Dependency injection | Method parameters | Constructor / field |
| Control | High | Medium |
| Best use | Third-party classes | Application classes |

# 7. Real World Example

Architecture:

```
Repository → Service → Controller
```

Configuration:

```java
@Configuration
public class AppConfig {

    @Bean
    public UserRepository userRepository() {
        return new UserRepository();
    }

    @Bean
    public UserService userService(UserRepository userRepository) {
        return new UserService(userRepository);
    }
}
```

Dependency wiring:

```
UserRepository → UserService
```

---

# 8. Important Rule

Spring **automatically resolves method parameters from the container**.

Example:

```java
@Bean
public OrderService orderService(PaymentService paymentService)
```

Spring injects the `PaymentService` bean automatically.

---

# 9. Internal Optimization with `@Configuration`

Spring uses **CGLIB proxy** for configuration classes.

Example:

```java
@Bean
public Car car() {
    return new Car(engine());
}
```

Even if `engine()` is called multiple times, Spring ensures **only one Engine bean is created (Singleton)**.

```
Complex object creation
```

# 11. Multiple Beans of Same Type Problem

If **multiple beans of the same type exist**, Spring cannot decide which bean to inject.

Spring throws:

```
NoUniqueBeanDefinitionException
```

---

# 12. Example with Multiple Beans

```java
@Configuration
public class AppConfig {

    @Bean
    public Engine petrolEngine() {
        return new Engine("Petrol");
    }

    @Bean
    public Engine dieselEngine() {
        return new Engine("Diesel");
    }

    @Bean
    public Car car(Engine engine) {
        return new Car(engine);
    }
}
```

Spring finds:

```
Engine candidates:
1. petrolEngine
2. dieselEngine
```

Result:

```
NoUniqueBeanDefinitionException
No qualifying bean of type 'Engine'
expected single matching bean but found 2
```

---

# 13. Solution 1 — Using `@Qualifier`

Specify which bean should be injected.

```java
@Bean
public Car car(@Qualifier("petrolEngine") Engine engine) {
    return new Car(engine);
}
```

Injection result:

```
petrolEngine → Car
```

---

# 14. Solution 2 — Using `@Primary`

Mark one bean as default.

```java
@Bean
@Primary
public Engine petrolEngine() {
    return new Engine("Petrol");
}

@Bean
public Engine dieselEngine() {
    return new Engine("Diesel");
}
```

Now Spring injects:

```
petrolEngine
```

---

# 15. Solution 3 — Inject by Bean Name

Spring can match **method parameter name with bean name**.

```java
@Bean
public Car car(Engine petrolEngine) {
    return new Car(petrolEngine);
}
```

Spring injects:

```
petrolEngine
```

---

# 16. Summary

| Case | Result |
|-----|------|
| One bean of type | Injected automatically |
| Multiple beans of same type | `NoUniqueBeanDefinitionException` |
| Use `@Qualifier` | Specify exact bean |
| Use `@Primary` | Default bean |
| Match parameter name | Inject by name |

# Interview One-Liner

Method-based bean wiring is the process of creating and injecting Spring beans using `@Bean` methods inside a `@Configuration` class, where dependencies are automatically resolved through method parameters by the Spring container.

## Practice code:

Vehicle.java

```java
package com.spring.demo.beans;

public class Vehicle {
    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Vehicle{" +
                "name='" + name + '\'' +
                '}';
    }
}

```

Main.java:

```java

import com.spring.demo.beans.Person;
import com.spring.demo.config.AppConfig;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

    var context = new AnnotationConfigApplicationContext(AppConfig.class);
```
