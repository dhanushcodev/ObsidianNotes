# BeanRegistrar in Spring Boot (Correct Usage)

## 1. What is BeanRegistrar?

`BeanRegistrar` is an interface used for **programmatic bean registration** in the Spring container.

Instead of using annotations like:

- `@Component`
- `@Bean`
- `@Configuration`

you can **manually register beans during application startup**.

It provides **low-level control over how beans are created and registered**.

---

# 2. When is BeanRegistrar Used?

`BeanRegistrar` is useful when:

- Beans must be created **dynamically**
- Bean creation depends on **runtime conditions**
- You are building **frameworks or libraries**
- You want **full control over bean registration**

---

# 3. BeanRegistrar Interface

```java
public interface BeanRegistrar {
    void register(BeanRegistry registry);
}
```

Spring calls this method during **application context initialization**.

---

# 4. Important Note

`spec.constructorArgs()` **does NOT exist** in the Spring API.

The **correct way to pass constructor arguments** is by using a **Supplier**.

---

# 5. Correct Way to Register Beans

The correct way to register beans with dependencies is:

```java
registry.registerBean(Class, Supplier)
```

The `Supplier` is responsible for **creating the object manually**.

---

# 6. Example

## Step 1: Engine Class

```java
public class Engine {

    public void start() {
        System.out.println("Engine started");
    }
}
```

---

## Step 2: Car Class

```java
public class Car {

    private final Engine engine;

    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
        System.out.println("Car driving");
    }
}
```

---

## Step 3: BeanRegistrar Implementation

```java
import org.springframework.beans.factory.BeanRegistrar;
import org.springframework.beans.factory.BeanRegistry;

public class CarBeanRegistrar implements BeanRegistrar {

    @Override
    public void register(BeanRegistry registry) {

        registry.registerBean(Engine.class);

        registry.registerBean(
            Car.class,
            () -> new Car(registry.getBean(Engine.class))
        );
    }
}
```

Explanation:

- `registry.registerBean(Engine.class)` registers the `Engine` bean.
- `registry.registerBean(Car.class, Supplier)` creates the `Car` object.
- `registry.getBean(Engine.class)` injects the dependency.

---

## Step 4: Configuration Class

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Import;

@Configuration
@Import(CarBeanRegistrar.class)
public class AppConfig {
}
```

`@Import` tells Spring to **load the BeanRegistrar**.

---

## Step 5: Main Application

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {

    public static void main(String[] args) {

        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        Car car = context.getBean(Car.class);
        car.drive();
    }
}
```

---

# 7. Output

```
Engine started
Car driving
```

# 8. Common registerBean Methods

## Register Bean by Type

```java
registry.registerBean(MyService.class);
```

Spring will automatically create the bean.

---

## Register Bean with Supplier

```java
registry.registerBean(
    MyService.class,
    () -> new MyService()
);
```

Used when you want **manual object creation**.

---

## Register Bean with Dependency

```java
registry.registerBean(
    PaymentService.class,
    () -> new PaymentService(registry.getBean(Database.class))
);
```

The dependency is fetched from the container.

---

## Register Bean with Name

```java
registry.registerBean("myService", MyService.class);
```

This explicitly defines the **bean name**.

---

# 9. Advantages of BeanRegistrar

- Allows **programmatic bean registration**
- Supports **dynamic bean creation**
- Useful for **framework development**
- Provides **fine-grained control over bean creation**
- Can register beans **based on runtime conditions**

---

# 10. When Should You Use BeanRegistrar?

Use `BeanRegistrar` when:

- You need **dynamic bean creation**
- You are building **Spring extensions or frameworks**
- Bean creation depends on **runtime logic**

Avoid using it for normal applications because:

- `@Component`
- `@Bean`
- `@Configuration`

are simpler and more readable.
