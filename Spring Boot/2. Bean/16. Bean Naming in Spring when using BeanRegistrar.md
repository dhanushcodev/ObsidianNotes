# Bean Naming in Spring when using BeanRegistrar

## Problem Scenario

Example code using `BeanRegistrar`:

```java
public class CarBeanRegistrar implements BeanRegistrar {

    @Override
    public void register(BeanRegistry registry, Environment env) {

        registry.registerBean(Vehicle.class);

        registry.registerBean(Person.class, personSpec ->
                personSpec.supplier(ctx -> {
                    Person person = new Person(ctx.bean(Vehicle.class));
                    person.setName("Test person");
                    return person;
                })
        );
    }
}
```

Configuration class:

```java
@Configuration
@Import(CarBeanRegistrar.class)
public class AppConfig {
}
```

Main class:

```java
var context = new AnnotationConfigApplicationContext(AppConfig.class);

if(context.containsBean("person")){
    Person person = (Person) context.getBean("person");
}

if(context.containsBean("vehicle")){
    Vehicle vehicle = (Vehicle) context.getBean("vehicle");
}
```

Both `if` conditions return **false**.

---

# Why This Happens

## Default Bean Naming Rule in BeanRegistrar

When registering beans using:

```java
registry.registerBean(Class)
```

Spring does **NOT** use the simple class name as the bean name.

Instead, Spring uses the **Fully Qualified Class Name (FQCN)**.

Example:

```java
registry.registerBean(Vehicle.class);
```

Bean name becomes:

```
com.spring.demo.beans.Vehicle
```

Example:

```java
registry.registerBean(Person.class, ...)
```

Bean name becomes:

```
com.spring.demo.beans.Person
```

---

# Why `containsBean("person")` Returns False

Your code checks:

```java
context.containsBean("person")
```

But the actual bean name inside Spring is:

```
com.spring.demo.beans.Person
```

Since `"person"` does not match the real bean name, the result is:

```
false
```

Same issue occurs for `"vehicle"`.

---

# How to Verify Actual Bean Names

You can print all beans in the Spring container.

```java
for(String name : context.getBeanDefinitionNames()) {
    System.out.println(name);
}
```

Example output:

```
appConfig
com.spring.demo.beans.Vehicle
com.spring.demo.beans.Person
```

# Correct Ways to Access Beans

## 1. Access Beans by Type (Recommended)

Spring encourages **type-based lookup**.

```java
Person person = context.getBean(Person.class);
Vehicle vehicle = context.getBean(Vehicle.class);
```

Advantages:

- safer
- refactor friendly
- avoids naming mistakes

---

## 2. Access Beans Using Generated Bean Name

```java
Person person = (Person) context.getBean("com.spring.demo.beans.Person");
Vehicle vehicle = (Vehicle) context.getBean("com.spring.demo.beans.Vehicle");
```

---

## 3. Register Beans with Explicit Names (Best Practice)

Define bean names manually during registration.

```java
registry.registerBean("vehicle", Vehicle.class);

registry.registerBean("person", Person.class,
        spec -> spec.supplier(ctx -> {
            Person person = new Person(ctx.bean(Vehicle.class));
            person.setName("Test person");
            return person;
        })
);
```

Now the bean names will be:

```
vehicle
person
```

Your code will work:

```java
if(context.containsBean("person")){
    Person person = context.getBean(Person.class);
}

if(context.containsBean("vehicle")){
    Vehicle vehicle = context.getBean(Vehicle.class);
}
```

# Bean Naming Summary

| Registration Method | Bean Name |
|---|---|
| `registerBean(Class)` | Fully Qualified Class Name |
| `registerBean("name", Class)` | Custom Name |
| `@Component` | decapitalized class name |
| `@Bean` | method name |

Example:

| Bean | Default Name |
|---|---|
| Vehicle.class | com.spring.demo.beans.Vehicle |
| Person.class | com.spring.demo.beans.Person |

---

# Best Practice

Prefer **type-based lookup** instead of name-based lookup.

Good:

```java
context.getBean(Person.class);
```

Avoid when possible:

```java
context.getBean("person");
```

Reasons:

- avoids naming issues
- easier refactoring
- safer dependency resolution
