# @Primary and @Qualifier in Spring Autowiring

## 1. `@Primary` in Spring Autowiring

When multiple beans of the same type exist in the Spring container, Spring does not know which one to inject. This results in **NoUniqueBeanDefinitionException**.

`@Primary` tells Spring:

> If multiple beans exist, use this bean as the default.

### Example

```java
interface Engine {
    void start();
}

@Component
class PetrolEngine implements Engine {
    public void start() {
        System.out.println("Petrol Engine Started");
    }
}

@Component
@Primary
class DieselEngine implements Engine {
    public void start() {
        System.out.println("Diesel Engine Started");
    }
}
```

```java
@Component
class Car {

    private Engine engine;

    @Autowired
    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
    }
}
```

### What Happens Here

Two beans exist:

- `PetrolEngine`
- `DieselEngine`

Because `DieselEngine` is marked with `@Primary`, Spring injects it automatically.

### Output

```
Diesel Engine Started
```

### Key Points

- Used when **one bean should be the default choice**
- Works automatically during **autowiring**
- Usually only **one bean should be marked `@Primary`**

# 2. `@Qualifier` in Spring Autowiring

`@Qualifier` is used when you want to **explicitly select a specific bean**.

Instead of letting Spring decide, you tell Spring exactly which bean to inject.

### Example

```java
interface Engine {
    void start();
}

@Component("petrolEngine")
class PetrolEngine implements Engine {
    public void start() {
        System.out.println("Petrol Engine Started");
    }
}

@Component("dieselEngine")
class DieselEngine implements Engine {
    public void start() {
        System.out.println("Diesel Engine Started");
    }
}
```

```java
@Component
class Car {

    private Engine engine;

    @Autowired
    public Car(@Qualifier("petrolEngine") Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
    }
}
```

### Output

```
Petrol Engine Started
```

Spring injects the bean named `"petrolEngine"`.

---

# 3. `@Primary` vs `@Qualifier`

| Feature | `@Primary` | `@Qualifier` |
|--------|------------|-------------|
| Purpose | Default bean selection | Explicit bean selection |
| Usage Location | On bean class | On injection point |
| When Used | Multiple beans exist | Need a specific bean |
| Control Level | Global default | Precise control |

---

# 4. When Both Are Used

If both are present, **`@Qualifier` takes priority**.

### Example

```java
@Component
@Primary
class DieselEngine implements Engine {}

@Component
class PetrolEngine implements Engine {}
```

```java
@Autowired
public Car(@Qualifier("petrolEngine") Engine engine) {
    this.engine = engine;
}
```

Even though `DieselEngine` is marked as `@Primary`, Spring injects `PetrolEngine` because `@Qualifier` overrides `@Primary`.

---

# 5. Real-World Use Case

Example: Payment Systems

```
PaymentService
 ├── CreditCardPayment
 ├── UpiPayment
 └── PaypalPayment
```

- Mark **UPI** as `@Primary` (default payment method)
- Use `@Qualifier` when specifically selecting **PayPal or Credit Card**

---

# Summary

- `@Primary` → Defines the **default bean** when multiple beans of the same type exist.
- `@Qualifier` → Specifies **exactly which bean to inject**.
- `@Qualifier` **overrides** `@Primary`.
