# Eager vs Lazy Initialization in Spring Boot

## 1. Bean Initialization in Spring

In Spring Boot, **beans are objects managed by the Spring IoC container**.  
Spring decides **when to create these beans**.

There are two approaches:

- **Eager Initialization**
- **Lazy Initialization**

---

# 2. Eager Initialization

## Definition
Eager initialization means the **bean is created when the Spring container starts**.

Spring Boot **creates singleton beans eagerly by default**.

## Example

```java
@Component
public class Car {

    public Car() {
        System.out.println("Car bean created");
    }
}
```

### Main Application

```java
@SpringBootApplication
public class App {
    public static void main(String[] args) {
        SpringApplication.run(App.class, args);
    }
}
```

### Output

```
Car bean created
```

Even if `Car` is never used, Spring still creates it during application startup.

---

## Advantages of Eager Initialization

### 1. Early Error Detection

If dependencies are missing, the application fails **during startup**.

Example:

```java
@Component
public class Car {

    public Car(Engine engine) {
    }
}
```

If `Engine` bean does not exist, the application will **fail immediately at startup**.

---

### 2. Faster Bean Access

Since beans are already created, they are **immediately available when needed**.

---

### 3. Suitable for Core Beans

Useful for important beans like:

- Database configuration
- Security services
- Core business services

---

## Disadvantages of Eager Initialization

### 1. Slower Application Startup

All beans are created when the application starts.

### 2. Higher Memory Usage

Even unused beans occupy memory.

---

# 3. Lazy Initialization

## Definition

Lazy initialization means the bean is **created only when it is first requested** from the Spring container.

---

## Using `@Lazy`

```java
@Component
@Lazy
public class Bike {

    public Bike() {
        System.out.println("Bike bean created");
    }
}
```

---

### Main Class

```java
@SpringBootApplication
public class App {

    public static void main(String[] args) {

        ConfigurableApplicationContext context =
                SpringApplication.run(App.class, args);

        Bike bike = context.getBean(Bike.class);
    }
}
```

### Output

```
Bike bean created
```

The bean is created **only when `getBean()` is called**.

---

# 4. Lazy Initialization with `@Bean`

```java
@Configuration
public class AppConfig {

    @Bean
    @Lazy
    public Engine engine() {
        return new Engine();
    }
}
```

---

# 5. Lazy Dependency Injection

Lazy can also be applied to dependencies.

```java
@Component
public class Car {

    private final Engine engine;

    public Car(@Lazy Engine engine) {
        this.engine = engine;
    }
}
```

Here, the `Engine` bean will be created **only when it is actually needed**.

---

# 6. Enable Lazy Initialization for Entire Application

Spring Boot allows enabling lazy initialization globally.

### application.properties

```
spring.main.lazy-initialization=true
```

Now **all beans become lazy by default**.

---

# 7. Eager vs Lazy Comparison

| Feature | Eager Initialization | Lazy Initialization |
|------|------|------|
| Bean Creation | At application startup | When first requested |
| Default Behavior | Yes (for singleton beans) | No |
| Startup Time | Slower | Faster |
| Runtime Access | Faster | Slight delay on first use |
| Error Detection | Early | Late |
| Memory Usage | Higher | Lower |

---

# 8. Real World Examples

## Eager Beans

Used for:

- Database connection pool
- Security configuration
- Core services

Example:

```java
@Service
public class PaymentService {
}
```

---

## Lazy Beans

Used for:

- Optional features
- Heavy objects
- Rarely used services

Example:

```java
@Service
@Lazy
public class ReportGenerator {
}
```

Reports will be generated **only when required**.

---

# 9. When to Use Lazy Initialization

Use Lazy when:

- Bean creation is **expensive**
- Bean is **rarely used**
- You want **faster startup**
- You want to **avoid circular dependencies**

---

# 10. Important Interview Points

- Spring **singleton beans are eager by default**
- `@Lazy` makes beans **initialize only when needed**
- `@Lazy` can be used on:
  - Class
  - `@Bean` methods
  - Constructor parameters
- Global lazy initialization can be enabled using:

```
spring.main.lazy-initialization=true
```
