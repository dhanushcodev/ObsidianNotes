# `@DependsOn` in Spring Boot

## What is `@DependsOn`?

`@DependsOn` is a Spring annotation used to **control the initialization order of beans**.

It tells Spring that **one bean depends on another bean**, so the dependent bean should be **created only after the specified bean is initialized**.

Normally, Spring **does not guarantee the order of bean creation** unless there is a dependency.  
`@DependsOn` explicitly defines that order.

---

## Why Do We Use `@DependsOn`?

Sometimes a bean must be initialized **before another bean starts**, for example:

- Database initialization must happen before service beans
- Cache loading must happen before application services
- Configuration setup must run before other beans use it

In such cases, we use `@DependsOn`.

---

## Basic Syntax

```java
@DependsOn("beanName")
```

Or with multiple dependencies:

```java
@DependsOn({"bean1", "bean2"})
```

---

## Example 1: Basic Bean Dependency

### Database Initialization Bean

```java
@Component("database")
public class Database {

    public Database() {
        System.out.println("Database initialized");
    }
}
```

### Service Bean That Depends on Database

```java
@Component
@DependsOn("database")
public class UserService {

    public UserService() {
        System.out.println("UserService initialized");
    }
}
```

### Output

```
Database initialized
UserService initialized
```

Even if Spring scans `UserService` first, it will **initialize `Database` first**.

---

## Example 2: Multiple Dependencies

```java
@Component
@DependsOn({"database", "cacheManager"})
public class OrderService {

    public OrderService() {
        System.out.println("OrderService initialized");
    }
}
```

This ensures the initialization order:

```
database → cacheManager → orderService
```

---

## Example 3: Using `@DependsOn` with `@Bean`

```java
@Configuration
public class AppConfig {

    @Bean
    public Database database() {
        return new Database();
    }

    @Bean
    @DependsOn("database")
    public UserService userService() {
        return new UserService();
    }
}
```

---

## Important Notes

1. `@DependsOn` **only controls initialization order**, not dependency injection.
2. It **does not automatically inject the dependency**.
3. It is mostly used when **there is no direct dependency but order matters**.

Example:

```java
@DependsOn("cacheLoader")
```

Even if the bean **doesn't use `cacheLoader` directly**, it will still wait for it.

---

## Difference Between `@Autowired` and `@DependsOn`

| Feature | `@Autowired` | `@DependsOn` |
|--------|-------------|-------------|
| Purpose | Inject dependency | Control initialization order |
| Creates dependency | Yes | No |
| Used for | Object injection | Startup sequencing |

Example:

```java
@Autowired
private Database database;
```

vs

```java
@DependsOn("database")
```

---

## When Should You Use `@DependsOn`?

Use it when:

- You need **startup order control**
- There is **no direct dependency**
- A bean must **run initialization logic before others**

Common examples:

- Logging initialization
- Cache loading
- Database migration
- Configuration setup

---

## When NOT to Use It

Avoid `@DependsOn` if:

- You can use **constructor injection**
- Beans are already **directly dependent**

Better approach example:

```java
@Service
public class UserService {

    private final Database database;

    public UserService(Database database) {
        this.database = database;
    }
}
```

Spring will **automatically initialize `Database` first**.

---

## Key Takeaway

`@DependsOn` is used when **bean initialization order matters but there is no direct dependency between beans**.
