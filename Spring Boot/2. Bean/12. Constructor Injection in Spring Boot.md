# Constructor Injection in Spring Boot

## What is Constructor Injection
Constructor Injection is a type of Dependency Injection where Spring injects required dependencies through the **constructor of a class**.

When Spring creates a bean, it calls the constructor and provides the dependencies from the **Spring Application Context**.

---

# Advantages of Constructor Injection

## 1. Ensures Mandatory Dependencies
Constructor injection ensures that required dependencies must be provided when the object is created.

If Spring cannot find the dependency, the application **fails during startup instead of runtime**.

### Example

```java
@Component
class Engine {
    public void start() {
        System.out.println("Engine Started");
    }
}
```

```java
@Component
class Car {

    private final Engine engine;

    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
        System.out.println("Car Driving");
    }
}
```

If the `Engine` bean is missing, Spring throws:

```
NoSuchBeanDefinitionException
```

This prevents runtime errors.

### Important Note (Spring Boot 4.3+)

If a class has **only one constructor**, `@Autowired` is **not required**.

```java
@Component  
public class Car {  
  
    private final Engine engine;  
  
    public Car(Engine engine) {  
        this.engine = engine;  
    }  
  
}
```

Spring will automatically inject the dependency.

---

## 2. Supports Immutability
Constructor injection allows dependencies to be declared as **final**, making them immutable after object creation.

This improves **code safety and design**.

### Example

```java
@Component
class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }

}
```

Since the dependency is `final`, it **cannot be modified later**.

Bad example using field injection:

```java
@Autowired
private PaymentService paymentService;
```

This makes the object **mutable**.

---

## 3. Better for Unit Testing
Constructor injection allows dependencies to be **manually injected without starting the Spring container**.

### Example

```java
class PaymentService {
    public void pay() {
        System.out.println("Payment Done");
    }
}
```

```java
class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }

    public void placeOrder() {
        paymentService.pay();
    }
}
```

### Unit Test Example

```java
public class OrderServiceTest {

    public static void main(String[] args) {

        PaymentService mockService = new PaymentService();

        OrderService orderService = new OrderService(mockService);

        orderService.placeOrder();
    }
}
```

No Spring container is required, making testing **faster and simpler**.

---

## 4. Prevents Null Dependencies
Constructor injection ensures dependencies are **initialized during object creation**.

### Example

```java
class UserService {

    private final EmailService emailService;

    public UserService(EmailService emailService) {
        this.emailService = emailService;
    }

}
```

Spring ensures `EmailService` exists before creating `UserService`.

With field injection:

```java
@Autowired
private EmailService emailService;
```

This can sometimes lead to:

```
NullPointerException
```

Constructor injection reduces this risk.

---

## 5. Makes Dependencies Explicit (Better Design)
Constructor injection clearly shows **all dependencies required by a class**.

This improves **readability and maintainability**.

### Example

```java
@Service
class OrderService {

    private final PaymentService paymentService;
    private final NotificationService notificationService;
    private final InventoryService inventoryService;

    public OrderService(
            PaymentService paymentService,
            NotificationService notificationService,
            InventoryService inventoryService) {

        this.paymentService = paymentService;
        this.notificationService = notificationService;
        this.inventoryService = inventoryService;
    }
}
```

From the constructor we can clearly see that `OrderService` depends on:

- PaymentService  
- NotificationService  
- InventoryService  

---

## 6. Detects Design Problems Early
If a constructor has **too many dependencies**, it indicates that the class may have **too many responsibilities**.

### Example

```java
public Car(
    Engine engine,
    Wheel wheel,
    Fuel fuel,
    Driver driver,
    MusicSystem musicSystem,
    GPS gps,
    AirCondition ac
)
```

This suggests the class violates the **Single Responsibility Principle**.

Constructor injection helps developers **identify design issues early**.

---

## 7. Works Well with Lombok
Constructor injection works very well with **Lombok's `@RequiredArgsConstructor`**.

### Example

```java
@Service
@RequiredArgsConstructor
class OrderService {

    private final PaymentService paymentService;
    private final NotificationService notificationService;

}
```

Lombok automatically generates the constructor for the required fields.

Spring will automatically inject the dependencies.

---

# Summary

Constructor Injection is the **recommended dependency injection method in Spring** because it:

- Ensures required dependencies
- Supports immutability
- Improves unit testing
- Prevents null dependencies
- Makes dependencies explicit
- Helps detect poor design early
- Works well with Lombok

## Practice code:

Vehicle.class

```java
package com.spring.demo.beans;  
  
import org.springframework.beans.factory.annotation.Autowired;  
import org.springframework.stereotype.Component;  
  
@Component  
public class Person {  
  
    private String name;  
    private Vehicle vehicle;  
  
    public String getName() {  
        return name;  
    }  
      
    @Autowired  
    public Person(Vehicle vehicle) {  
        this.vehicle = vehicle;  
    }  
  
    public void setName(String name) {  
        this.name = name;  
    }  
  
    public Vehicle getVehicle() {  
        return vehicle;  
    }  
    public void setVehicle(Vehicle vehicle) {  
        this.vehicle = vehicle;  
    }  
}
```
