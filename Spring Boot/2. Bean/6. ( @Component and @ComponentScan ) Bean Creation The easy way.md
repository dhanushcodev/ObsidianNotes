# @Component, @ComponentScan and @SpringBootApplication in Spring

## 1. `@Component` Annotation

### What is `@Component`?

`@Component` is a **stereotype annotation** in Spring used to mark a class as a **Spring Bean**.

When Spring scans the project, it **automatically detects classes annotated with `@Component` and registers them in the Spring IoC Container**.

This process is called **Component Scanning**.

---

### Example

```java
import org.springframework.stereotype.Component;

@Component
public class Engine {

    public void start() {
        System.out.println("Engine Started");
    }
}
```

### What happens here?

1. Spring scans the package.
2. It finds the `Engine` class annotated with `@Component`.
3. Spring **creates an object (bean)** of the class.
4. The bean is **stored inside the Spring Container**.

So now we can retrieve it using:

```java
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

Engine engine = context.getBean(Engine.class);
engine.start();
```

Output:

```
Engine Started
```

---

### Custom Bean Name

By default, Spring uses the **class name with camelCase** as the bean name.

Example:

```java
@Component
public class Engine {}
```

Bean name will be:

```
engine
```

You can also give a **custom name**:

```java
@Component("myEngine")
public class Engine {}
```

Access using:

```java
Engine engine = context.getBean("myEngine", Engine.class);
```

---

### Real-Time Example

Example: **Food Delivery System**

```java
@Component
public class PaymentService {

    public void pay() {
        System.out.println("Payment Successful");
    }
}
```

Spring automatically manages the **PaymentService** object.

---

# 2. `@ComponentScan` Annotation

### What is `@ComponentScan`?

`@ComponentScan` tells Spring **where to search for `@Component` classes**.

Spring will **scan the specified package and automatically create beans** for all classes annotated with:

- `@Component`
- `@Service`
- `@Repository`
- `@Controller`

(All of these are internally `@Component`).

---

### Example

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
}
```

### What happens here?

Spring will scan the package:

```
com.example
```

And create beans for all classes annotated with `@Component`.

---

### Project Structure Example

```
com.example
 ├── AppConfig
 ├── service
 │     └── PaymentService
 └── repository
       └── OrderRepository
```

PaymentService:

```java
@Component
public class PaymentService {
}
```

OrderRepository:

```java
@Component
public class OrderRepository {
}
```

Config Class:

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
}
```

Now Spring automatically creates beans for:

```
PaymentService
OrderRepository
```

---

### Multiple Package Scanning

You can scan **multiple packages**.

```java
@ComponentScan({
    "com.example.service",
    "com.example.repository"
})
```

---

### Example with Main Class

```java
public class MainApp {

    public static void main(String[] args) {

        ApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        PaymentService payment = context.getBean(PaymentService.class);
        payment.pay();
    }
}
```

---

## Internal Flow (Important for Interviews)

1. Spring starts.
2. `@ComponentScan` tells Spring which package to scan.
3. Spring searches for classes annotated with `@Component`.
4. Spring creates objects of those classes.
5. Objects are stored in the **Spring IoC Container**.
6. Beans can be retrieved using `getBean()` or `@Autowired`.

---

## Relationship Between Them

| Annotation | Purpose |
|---|---|
| `@Component` | Marks a class as a Spring Bean |
| `@ComponentScan` | Tells Spring where to find those beans |

Simple rule:

```
@Component → Marks the bean
@ComponentScan → Finds the bean
```

---

# Why Are We Still Using `AppConfig`?

In **Spring Core (without Spring Boot)**, we need a **configuration class** to tell Spring:

- where to scan components  
- how to configure beans  
- how to initialize the container  

That configuration class is usually called **`AppConfig`**.

---

### Example

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
}
```

Then we start Spring like this:

```java
ApplicationContext context =
        new AnnotationConfigApplicationContext(AppConfig.class);
```

---

### What `AppConfig` Does

`AppConfig` acts like the **main configuration file** of Spring.

It tells Spring:

1. **Where to scan for beans**
2. **What beans to create**
3. **How the application should be configured**

Think of it like a **central instruction file** for the Spring container.

---

### Example Project (Spring Core)

```
com.example
 ├── AppConfig
 ├── service
 │     └── PaymentService
 └── repository
       └── OrderRepository
```

`PaymentService.java`

```java
@Component
public class PaymentService {
    public void pay() {
        System.out.println("Payment Done");
    }
}
```

`AppConfig.java`

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
}
```

`Main.java`

```java
public class Main {

    public static void main(String[] args) {

        ApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        PaymentService service = context.getBean(PaymentService.class);
        service.pay();
    }
}
```

---

# Why We Don't Usually Use `AppConfig` in Spring Boot

In **Spring Boot**, configuration is **automatically handled**.

Instead of writing `AppConfig`, Spring Boot uses:

```
@SpringBootApplication
```

as the **main configuration class**.

---

# `@SpringBootApplication` Explained

`@SpringBootApplication` is the **main annotation used to start a Spring Boot application**.

Example:

```java
@SpringBootApplication
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

This class replaces the need for **manual configuration like `AppConfig`**.

---

## What Happens When Spring Boot Starts

When you run:

```java
SpringApplication.run(MyApplication.class, args);
```

Spring Boot:

1. Starts the **Spring container**
2. Performs **component scanning**
3. Creates **all beans**
4. Applies **auto configuration**
5. Starts the **embedded server (Tomcat)**

---

# `@SpringBootApplication` Internally

`@SpringBootApplication` is actually a **combination of three annotations**.

```
@SpringBootApplication
```

Internally equals:

```
@Configuration
@EnableAutoConfiguration
@ComponentScan
```

---

## 1. `@Configuration`

Marks the class as a **Spring configuration class**.

Example:

```java
@Configuration
public class AppConfig {

    @Bean
    public Engine engine() {
        return new Engine();
    }
}
```

It allows defining **beans using `@Bean` methods**.

---

## 2. `@EnableAutoConfiguration`

This is the **most powerful feature of Spring Boot**.

Spring Boot automatically configures things like:

- Tomcat server
- DataSource
- Hibernate
- Spring MVC
- Jackson

Example:

If dependency is present:

```
spring-boot-starter-web
```

Spring Boot automatically configures:

```
DispatcherServlet
Tomcat
Spring MVC
JSON Converter
```

You don't need to configure them manually.

---

## 3. `@ComponentScan`

Spring Boot automatically scans components starting from the **package of the main class**.

Example:

```
com.example
 ├── MyApplication
 ├── controller
 ├── service
 └── repository
```

Spring Boot scans everything under:

```
com.example
```

So classes annotated with:

```
@Component
@Service
@Repository
@Controller
```

are automatically registered as beans.

---

# Real Project Example

```
com.example.foodapp
 ├── FoodApplication
 ├── controller
 │     └── OrderController
 ├── service
 │     └── OrderService
 └── repository
       └── OrderRepository
```

Main class:

```java
@SpringBootApplication
public class FoodApplication {

    public static void main(String[] args) {
        SpringApplication.run(FoodApplication.class, args);
    }
}
```

No need for:

```
AppConfig
@ComponentScan
XML configuration
Manual bean configuration
```

Spring Boot handles everything automatically.

---

# Simple Comparison

| Feature | Spring Core | Spring Boot |
|---|---|---|
| Configuration class | `AppConfig` | Main class |
| Component scanning | `@ComponentScan` manually | Automatic |
| Bean configuration | Manual | Mostly automatic |
| Server setup | Manual | Embedded Tomcat |
| Startup | `AnnotationConfigApplicationContext` | `SpringApplication.run()` |

---

# Easy Way to Remember

### Spring Core

You configure everything manually.

### Spring Boot

Spring configures everything automatically.

# Basic Working Example of `@SpringBootApplication`

This example shows a **minimal Spring Boot application** using `@SpringBootApplication`, `@RestController`, and a simple API.

---

## 1. Project Structure

```
com.example.demo
 ├── DemoApplication.java
 └── HelloController.java
```

---

## 2. Main Class (`@SpringBootApplication`)

`DemoApplication.java`

```java
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }

}
```

### What happens here?

`@SpringBootApplication` automatically enables:

```
@Configuration
@EnableAutoConfiguration
@ComponentScan
```

`SpringApplication.run()`:

1. Starts the **Spring container**
2. Performs **component scanning**
3. Creates all **Spring Beans**
4. Starts the **embedded Tomcat server**

---

## 3. Create a Controller

`HelloController.java`

```java
package com.example.demo;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello Spring Boot!";
    }

}
```

---

## 4. What Happens Internally

When the application starts:

1. Spring Boot scans the package

```
com.example.demo
```

2. It finds:

```
HelloController
```

3. Registers it as a **Spring Bean**

4. Maps the API:

```
GET /hello
```

---

## 5. Run the Application

Run the main class:

```java
SpringApplication.run(DemoApplication.class, args);
```

Spring Boot starts:

```
Tomcat server on port 8080
```

---

## 6. Test the API

Open browser:

```
http://localhost:8080/hello
```

Output:

```
Hello Spring Boot!
```

---

## 7. How Component Scanning Works

Because the main class is in:

```
com.example.demo
```

Spring Boot automatically scans:

```
com.example.demo.*
```

So it detects:

```
@RestController
@Service
@Repository
@Component
```

without needing `@ComponentScan` manually.

---

## 8. Why This Works Without Configuration

Spring Boot automatically configures:

```
Spring MVC
DispatcherServlet
Tomcat Server
JSON Conversion
```

Because of:

```
@EnableAutoConfiguration
```

---

## 9. Important Interview Point

`@SpringBootApplication` internally equals:

```java
@Configuration
@EnableAutoConfiguration
@ComponentScan
```

So it:

1. Defines configuration
2. Enables auto configuration
3. Scans components

---

## 10. Simplified Flow

```
Run Main Class
        ↓
Spring Boot Starts
        ↓
Component Scanning
        ↓
Beans Created
        ↓
Tomcat Server Starts
        ↓
API Available
```
