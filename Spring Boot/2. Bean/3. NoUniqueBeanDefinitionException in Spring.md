## Definition

`NoUniqueBeanDefinitionException` occurs in **Spring Framework / Spring Boot** when **more than one bean of the same type exists in the Spring container**, and Spring **does not know which bean to inject or return**.

In simple terms:

> Spring expected **one bean**, but **found multiple beans of the same type**.

This can happen in two scenarios:

1. **Dependency Injection (`@Autowired`)**
2. **Manual bean retrieval (`ApplicationContext.getBean()`)**

---

## 1. Why This Happens

Spring performs **dependency injection by type** by default.

If Spring finds **multiple beans of the same type**, it throws:

```
NoUniqueBeanDefinitionException
```

because it cannot decide which bean to use.

---

## 2. Example Setup

### Interface

```java
public interface PaymentService {
    void pay();
}
```

---

### Implementation 1

```java
@Component
public class CreditCardPaymentService implements PaymentService {

    @Override
    public void pay() {
        System.out.println("Paid using Credit Card");
    }
}
```

---

### Implementation 2

```java
@Component
public class UpiPaymentService implements PaymentService {

    @Override
    public void pay() {
        System.out.println("Paid using UPI");
    }
}
```

Now the Spring container contains **two beans of type `PaymentService`**:

- `creditCardPaymentService`
- `upiPaymentService`

---

## 3. Case 1: Exception During Dependency Injection

### Injection Example

```java
@Service
public class OrderService {

    @Autowired
    private PaymentService paymentService;

}
```

Spring tries to inject a bean of type:

```
PaymentService
```

But it finds two beans:

```
creditCardPaymentService
upiPaymentService
```

Spring cannot decide which one to inject.

### Exception

```
NoUniqueBeanDefinitionException:
No qualifying bean of type 'PaymentService' available:
expected single matching bean but found 2:
creditCardPaymentService, upiPaymentService
```

---

## 4. Case 2: Exception When Using getBean()

Spring can also throw this exception when retrieving beans manually.

### Example

```java
ApplicationContext context =
        new AnnotationConfigApplicationContext(AppConfig.class);

PaymentService service = context.getBean(PaymentService.class);
```

Spring searches for a bean of type:

```
PaymentService
```

But it finds two beans:

```
creditCardPaymentService
upiPaymentService
```

Since **more than one bean exists**, Spring throws:

```
NoUniqueBeanDefinitionException
```

---

## 5. Solutions

There are **three common solutions used in real projects**.

---

### Solution 1 — `@Primary`

Mark one bean as the **default bean**.

```java
@Component
@Primary
public class UpiPaymentService implements PaymentService {

    @Override
    public void pay() {
        System.out.println("Paid using UPI");
    }
}
```

Now Spring automatically selects:

```
UpiPaymentService
```

When this is called:

```java
context.getBean(PaymentService.class);
```

or

```java
@Autowired
PaymentService paymentService;
```

---

### Solution 2 — `@Qualifier` (Most Common in Real Projects)

Specify **which bean to inject**.

```java
@Service
public class OrderService {

    @Autowired
    @Qualifier("creditCardPaymentService")
    private PaymentService paymentService;

}
```

Now Spring injects:

```
CreditCardPaymentService
```

---

### Solution 3 — Get Bean by Name

When using `getBean()`, specify the bean name.

```java
PaymentService service =
        context.getBean("creditCardPaymentService", PaymentService.class);
```

Now Spring knows exactly which bean to return.

---

### Solution 4 — Inject or Retrieve All Beans

You can inject all implementations using `List` or `Map`.

### Using List

```java
@Autowired
private List<PaymentService> paymentServices;
```

Spring injects:

```
[CreditCardPaymentService, UpiPaymentService]
```

---

### Using Map

```java
Map<String, PaymentService> beans =
        context.getBeansOfType(PaymentService.class);
```

Output:

```
{
 creditCardPaymentService = CreditCardPaymentService
 upiPaymentService = UpiPaymentService
}
```

You can then choose which implementation to use.

---

## 6. Important Rules

### getBean() Behavior

| Method | Behavior |
|------|------|
| `getBean(Class<T>)` | Works only if **one bean of that type exists** |
| `getBean(String name)` | Fetches bean by **name** |
| `getBean(String name, Class<T>)` | Fetches bean by **name and type** |
| `getBeansOfType(Class<T>)` | Returns **all beans of that type** |

## 7. Real World Example

Consider an **e-commerce checkout system**.

Payment methods:

- Credit Card
- UPI
- Wallet
- Net Banking

All implement:

```java
PaymentService
```

Beans inside Spring container:

```
CreditCardPaymentService
UpiPaymentService
WalletPaymentService
```

Calling:

```java
context.getBean(PaymentService.class);
```

Spring cannot determine which payment method to return.

So it throws:

```
NoUniqueBeanDefinitionException
```

Correct approach:

```
context.getBean("upiPaymentService")
```

or use:

```
@Qualifier
```

## 9. Interview One-Line Answer

> `NoUniqueBeanDefinitionException` occurs when Spring tries to inject or retrieve a bean by type but finds multiple beans of the same type in the application context, making it unable to determine which bean to use.

Common fixes:

- `@Primary`
- `@Qualifier`
- Retrieving bean by name
- Injecting `List` or `Map` of beans
