# Ways to Create Beans in Spring (Detailed Explanation)

In **Spring Core**, a **Bean** is simply an **object whose lifecycle is managed by the Spring container**.

Example in normal Java:

```java
UserService userService = new UserService();
```

Normally we create objects using `new`.  
But in Spring, the **Spring container creates and manages these objects**, and those objects are called **Spring Beans**.

There are **4 main ways to create beans in Spring**.

---

# 1. Using `@Component` (Component Scanning)

This is the **most common method** in modern Spring applications.

Spring automatically scans the classpath and creates beans for classes annotated with `@Component`.
## Example

```java
@Component
public class Engine {

    public void start() {
        System.out.println("Engine started");
    }
}
```

Another class:

```java
@Component
public class Car {

    private Engine engine;

    @Autowired
    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
        System.out.println("Car driving");
    }
}
```

Configuration class:

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
}
```

Running the application:

```java
ApplicationContext context =
        new AnnotationConfigApplicationContext(AppConfig.class);

Car car = context.getBean(Car.class);
car.drive();
```

## What Spring Does Internally

1. Spring scans the package `com.example`
2. Finds classes annotated with `@Component`
3. Creates objects of those classes
4. Stores them inside the **IoC container**
5. Injects dependencies using `@Autowired`

## Real-world usage

```
@Controller
@Service
@Repository
```

All of these are **special types of `@Component`**.

Example:

```java
@Controller
class UserController {}

@Service
class UserService {}

@Repository
class UserRepository {}
```

Spring automatically creates beans for them.

---

# 2. Using `@Bean` (Manual Bean Creation)

This is used when:

- You **cannot modify the class**
- The class belongs to a **third-party library**

Example class (third-party):

```java
public class PaymentGateway {

    public void process() {
        System.out.println("Payment processed");
    }
}
```

Configuration class:

```java
@Configuration
public class AppConfig {

    @Bean
    public PaymentGateway paymentGateway() {
        return new PaymentGateway();
    }
}
```

Using the bean:

```java
ApplicationContext context =
        new AnnotationConfigApplicationContext(AppConfig.class);

PaymentGateway gateway = context.getBean(PaymentGateway.class);
gateway.process();
```

## What Happens Internally

Spring:

1. Reads the `@Configuration` class
2. Finds methods annotated with `@Bean`
3. Executes those methods
4. Stores the returned object in the container

The **bean name is the method name**.

Example:

```
@Bean
public PaymentGateway paymentGateway()
```

Bean name → `paymentGateway`

---

# 3. Using XML Configuration (Old Method)

Before annotations existed, Spring used **XML configuration**.

## Bean class

```java
public class Engine {

    public void start() {
        System.out.println("Engine started");
    }
}
```

## XML configuration

File: `beans.xml`

```xml
<beans>

    <bean id="engine" class="com.example.Engine"/>

</beans>
```

Loading the container:

```java
ApplicationContext context =
        new ClassPathXmlApplicationContext("beans.xml");

Engine engine = context.getBean("engine", Engine.class);
engine.start();
```

## What Spring Does

1. Reads the XML file
2. Creates objects defined in `<bean>`
3. Stores them in the container

This approach is **rarely used today**.

---

# 4. Using `@Configuration` + `@Bean` (Java Configuration)

This is the **modern replacement for XML configuration**.

Example:

```java
@Configuration
public class AppConfig {

    @Bean
    public Engine engine() {
        return new Engine();
    }

    @Bean
    public Car car() {
        return new Car(engine());
    }
}
```

Class definitions:

```java
public class Engine {
}

public class Car {

    private Engine engine;

    public Car(Engine engine) {
        this.engine = engine;
    }
}
```

Running the application:

```java
ApplicationContext context =
        new AnnotationConfigApplicationContext(AppConfig.class);

Car car = context.getBean(Car.class);
```

Spring automatically injects the `Engine` bean into `Car`.

---

# Comparison of Bean Creation Methods

| Method | When to Use | Example |
|------|-------------|--------|
| `@Component` | Your own classes | Services, Controllers |
| `@Bean` | Third-party classes | Redis, Kafka, SDKs |
| XML Configuration | Legacy applications | Old Spring projects |
| Java Config (`@Configuration`) | Manual bean configuration | Complex beans |

---

# Real Project Example

Typical **Spring Boot project structure**:

```
Controller Layer
Service Layer
Repository Layer
```

Example:

```java
@RestController
class UserController {}

@Service
class UserService {}

@Repository
class UserRepository {}
```

Spring Boot automatically **creates beans using component scanning**.

# Interview Answer

**Ways to create beans in Spring**

1. Using `@Component` annotation (component scanning)
2. Using `@Bean` annotation
3. Using XML configuration
4. Using Java configuration (`@Configuration` class)
