# Alternatives to `@PostConstruct` and `@PreDestroy`

In modern Spring (especially **Spring Boot 3+** and **Spring Framework 6**),  
`@PostConstruct` and `@PreDestroy` are **no longer recommended** because they come from **`javax.annotation`**, which was removed from the JDK and moved to **Jakarta EE**.

Spring now recommends **Spring-native alternatives**.

---

# 1. Using `InitializingBean` and `DisposableBean`

Spring provides two interfaces for lifecycle callbacks.

| Interface | Method | Purpose |
|---|---|---|
| `InitializingBean` | `afterPropertiesSet()` | Runs after bean initialization |
| `DisposableBean` | `destroy()` | Runs before bean destruction |

## Example

```java
import org.springframework.beans.factory.DisposableBean;
import org.springframework.beans.factory.InitializingBean;
import org.springframework.stereotype.Component;

@Component
public class DatabaseConnection implements InitializingBean, DisposableBean {

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("Bean initialized: Opening database connection");
    }

    @Override
    public void destroy() throws Exception {
        System.out.println("Bean destroyed: Closing database connection");
    }
}
```

### Execution

```
Bean initialized: Opening database connection
Bean destroyed: Closing database connection
```

---

# 2. Using `initMethod` and `destroyMethod` (Recommended)

You can define lifecycle methods directly in the **configuration class**.

## Example

```java
public class DatabaseConnection {

    public void init() {
        System.out.println("Opening DB connection");
    }

    public void cleanup() {
        System.out.println("Closing DB connection");
    }
}
```

## Configuration Class

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {

    @Bean(initMethod = "init", destroyMethod = "cleanup")
    public DatabaseConnection databaseConnection() {
        return new DatabaseConnection();
    }
}
```

### Output

```
Opening DB connection
Closing DB connection
```

### Advantages

- No dependency on annotations
- Pure Spring configuration
- Recommended for **Spring Boot applications**

---

# 3. Using `@Bean` + Lambda Initialization (Spring Boot Style)

You can initialize inside the bean creation.

```java
@Bean
public DatabaseConnection databaseConnection() {
    DatabaseConnection db = new DatabaseConnection();
    db.init();
    return db;
}
```

---

# 4. Using `SmartLifecycle` (Advanced)

For **complex lifecycle control** (start/stop phases).

```java
import org.springframework.context.SmartLifecycle;
import org.springframework.stereotype.Component;

@Component
public class MyService implements SmartLifecycle {

    @Override
    public void start() {
        System.out.println("Service started");
    }

    @Override
    public void stop() {
        System.out.println("Service stopped");
    }

    @Override
    public boolean isRunning() {
        return true;
    }
}
```

Used in **advanced frameworks like messaging systems**.

# Comparison

| Feature | PostConstruct / PreDestroy | InitializingBean / DisposableBean | initMethod / destroyMethod |
|---|---|---|---|
| Type | Annotation | Interface | Configuration |
| Spring Dependency | No | Yes | No |
| Recommended | ❌ | ⚠️ | ✅ |
| Flexibility | Medium | Medium | High |

---

# Best Practice in Spring Boot

Most recommended approach:

```java
@Bean(initMethod="init", destroyMethod="cleanup")
```

Because it:

- avoids deprecated APIs
- keeps lifecycle management in configuration
- works well with **Spring Boot**.
