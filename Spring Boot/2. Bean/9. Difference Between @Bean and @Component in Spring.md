# Difference Between `@Bean` and `@Component` in Spring

Both **`@Bean`** and **`@Component`** are used in **Spring** to create beans and register them in the **Spring IoC container**, but they are used in different ways and scenarios.

---

# 1. `@Component`

## Definition
`@Component` is a **class-level annotation** used to tell Spring that the class is a **Spring-managed bean**.

Spring automatically detects these classes during **component scanning**.

## Example

```java
import org.springframework.stereotype.Component;

@Component
public class EmailService {

    public void sendEmail() {
        System.out.println("Sending email...");
    }
}
```

## How it gets registered

Spring scans packages using:

```java
@ComponentScan("com.example")
```

or automatically via:

```java
@SpringBootApplication
```

## Usage

```java
@Autowired
EmailService emailService;
```

## Key Points

- Applied on **classes**
- Uses **component scanning**
- Bean name = **class name (camelCase)** by default
- Used when **you control the class**

Example bean name:

```
EmailService → emailService
```

---

## Specialized Versions of `@Component`

Spring provides specialized stereotypes:

| Annotation | Purpose |
|---|---|
| `@Component` | Generic component |
| `@Service` | Business logic layer |
| `@Repository` | DAO layer |
| `@Controller` | Spring MVC controller |

All of them are internally **`@Component`**.

---

# 2. `@Bean`

## Definition

`@Bean` is a **method-level annotation** used inside a **configuration class** to manually create and configure beans.

## Example

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {

    @Bean
    public EmailService emailService() {
        return new EmailService();
    }
}
```

## Usage

```java
@Autowired
EmailService emailService;
```

## Key Points

- Applied on **methods**
- Used inside **`@Configuration` classes**
- Bean name = **method name**
- Used when **manual control is required**

Example bean name:

```
@Bean
public EmailService emailService()

bean name → emailService
```

---

# 3. Real World Scenario

## When to use `@Component`

When **you wrote the class** and want Spring to automatically detect it.

Example:

- Service classes
- Repository classes
- Controllers
- Utility classes

Example:

```java
@Service
public class PaymentService {}
```

---

## When to use `@Bean`

When the class:

- Comes from a **third-party library**
- Needs **custom initialization**
- Requires **complex configuration**

Example:

```java
@Bean
public ObjectMapper objectMapper() {
    ObjectMapper mapper = new ObjectMapper();
    mapper.enable(SerializationFeature.INDENT_OUTPUT);
    return mapper;
}
```

You **cannot add `@Component` to external classes**, so you create them using `@Bean`.

---

# 4. Key Differences

| Feature | `@Component` | `@Bean` |
|---|---|---|
| Annotation Type | Class-level | Method-level |
| Used In | Any class | Inside `@Configuration` class |
| Bean Creation | Automatic | Manual |
| Detection | Component scanning | Explicit declaration |
| Bean Name | Class name (camelCase) | Method name |
| Control Level | Less control | Full control |
| External Library Beans | Not possible | Possible |

---

# 5. Internal Working Difference

## `@Component`

Spring performs:

```
Classpath scanning
↓
Find classes with @Component
↓
Create object
↓
Register bean in container
```

---

## `@Bean`

Spring performs:

```
Load @Configuration class
↓
Execute @Bean method
↓
Return object
↓
Register bean in container
```

---

# 6. Simple Visualization

## `@Component`

```java
@Component
class Car {}
```

Spring automatically creates:

```
Car car = new Car();
```

---

## `@Bean`

```java
@Bean
Car car(){
   return new Car();
}
```

You **manually tell Spring how to create it**.

---

# 7. Interview One-Line Answer

> **`@Component` is used for automatic bean detection through component scanning, while `@Bean` is used to manually define and configure beans inside a `@Configuration` class.**
